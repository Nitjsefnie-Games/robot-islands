Server simulation — 10-iteration optimizer run

The server is authoritative: the entire pure simulation re-runs server-side on every intent (loadAndCatchUp) and every ~1 s periodic push (catchUp) — deserializeWorld → advanceWorldEconomy → advanceWorldSystems → serializeWorld. This run profiled that path on a real late-game save and applied behaviour-preserving optimizations under a byte-identical golden gate. Offline catch-up (1 h gap): 2.27 s → ~78 ms (~29×). Per-push (1 s gap): 8.85 ms → 7.15 ms. 10 iterations across two rounds — 7 kept, 3 reverted (1 broke a contract test, 2 measured sub-noise). Every kept change is byte-identical; the full 3571-test suite passes.

Iteration log

One optimization per iteration: profile → pick the biggest target → apply one change → re-benchmark + re-check the golden gate + targeted tests → keep only if it beats the noise floor and the gate is byte-identical, else revert. 5 iterations: 4 kept, 1 reverted.

iter 1Memoize islandTileCount KEPT · 1h 2.27s → 162ms
Target
islandTileCount — 50.9% of all CPU. findNextMerge rasterized every island's ellipse footprint (O(major×minor) + a Set of "x,y" keys) once per island per catch-up step (≈3600× for 1h), over static geometry.
Fix
Memoize on the spec; recompute only when live geometry changed. Exact field compare (not object identity — a merge mutates extraEllipses in place; not a hash — zero collision risk).
Gate
Golden byte-identical · 94 world/merge/reclamation tests pass. Killed >50% self-time + most GC churn.
iter 2Hoist antenna signal-range recompute out of the step loop KEPT · 1h 162 → 108ms
Target
computeSignalRanges — ~10% of catch-up. tickDrones rebuilt it (O(buildings)) every step; ranges are static except across a merge or vehicle settle.
Fix
advanceWorldSystems computes ranges once, recomputes only at those topology changes, passes via a new optional precomputedRanges param (client per-frame path unchanged). Reproduces the per-step result exactly.
Gate
Golden byte-identical · 209 drones/settlement/merge/antenna tests pass.
iter 3Skip skillMulSignature via the version counter REVERTED · broke a contract test
Target
skillMulSignature — 11.5% self. Route dispatch calls effectiveSkillMultipliers per route per step; the memo rebuilt the full signature each time. auraAmpVersion already bumps 1:1 with every unlockedNodes mutation, so a version match seemed to prove the signature unchanged.
Why reverted
The golden passed, but a unit test — “signature catches a direct Set add, no version bump” — failed: the signature is a deliberate safety net for direct unlockedNodes mutation that bypasses the version bump. The test suite is part of the gate, so: revert.
Lesson
The golden gate is necessary but not sufficient — targeted unit tests caught a documented contract the golden's 1h catch-up never exercises. “Behaviour identical” means all observable behaviour, not just the sampled path.
iter 4Content-key memo for islandCells KEPT · deserialize 2.4 → 2.0ms / push
Target
deserializeRevealedCells rasterizes every island's footprint via islandCells on every load — and the server re-loads on every push/intent.
Fix
Memoize on a content key (a WeakMap on the spec misses every load — fresh objects; a content key hits across loads of the same geometry). Capped + cleared wholesale; returns a fresh copy so callers may mutate.
Gate
Golden byte-identical · 154 discovery/persistence tests pass. Deserialize phase −18% on every server op.
iter 5Content-key islandTileCount — also covers the per-push case KEPT · 1s systems 0.70 → 0.16ms
Target
After iter 4, re-profiling the 1s push showed computeIslandTileCount back at 5% — iter 1's WeakMap (object identity) misses the single-step push because each deserialize builds fresh spec objects.
Fix
Switch to a content key (radii + extraEllipses; cx/cy and rotation not read — count is position-independent). Hits across pushes and keeps the catch-up benefit; supersedes iter 1's WeakMap and makes the in-place-mutation concern moot.
Gate
Golden byte-identical · 94 world/merge/reclamation tests pass. 1s-push systems phase 4.3× lighter.

Round 2 (iterations 6–10): a second pass at the user's request. The catch-up was already ~21×, so the remaining targets were route-dispatch costs (real) and the per-step weather sampling (mostly a benchmark cold-anchor artifact — not chased).

iter 6Hoist route-dispatch skill multipliers KEPT · 1h systems ~90 → ~85ms
Target
dispatchPhase called effectiveSkillMultipliers per route per step (rebuilds memo signature + clones).
Fix
advanceWorldSystems folds per-island mults ONCE (skills static in the loop) and threads them via a new precomputedSkillMul param. Preserves the signature contract iter 3 broke — it's still called, just far less. Falls back to a live fold on any map miss.
Gate
Golden byte-identical · 241 routes/skilltree tests pass.
iter 7Thread skill mults into route-cargo cap() KEPT · 1h systems ~85 → ~69ms
Target
A call-tree trace pinned ~11% of catch-up to effectiveSkillMultipliers ← cap ← destinationHeadroom ← tryPush: route-cargo planning called cap() per resource per route per step WITHOUT the skill-mult arg, re-folding skills.
Fix
planRouteCargo resolves src/dest mults once (from the iter-6 map, else one fold) and threads them into every cap(). Byte-identical — the value passed is exactly what cap() would fold.
Gate
Golden byte-identical · 100 routes tests pass. Biggest round-2 win.
iter 8Memoize route path-cell rasterization (lineSegmentCells) REVERTED · sub-noise
Target / fix
Route-path DDA rasterization recomputed per step over static endpoints — memoized on a content key.
Why reverted
Golden byte-identical, but 1h moved <2% — inside the 3% noise floor. The neighbouring advanceWalker was the cold-anchor artifact, not steady cost, and rasterizeLineSegment still maps the cells. A global cache for no measurable gain ⇒ discard.
iter 9Drop per-building conditional-spread allocations on load KEPT · deserialize ~1.97 → ~1.93ms
Target
deserializeWorld (the #1 per-push self-time) used ...(cond ? {f} : {}) ×3 per building — up to 546 throwaway objects per load (every push/intent).
Fix
Clone once, re-stamp the perf-domain timestamps in place. Byte-identical; deserialize-phase distributions non-overlapping (1.93–1.94 vs 1.96–2.01) — small but real, no added state.
Gate
Golden byte-identical · 119 persistence tests pass.
iter 10Memoize buildIslandIndex (id→spec, per step) REVERTED · sub-noise
Target / fix
Per-step id→spec index rebuild; memoized on (array ref + length), provably safe (islands only change via splice/push, never in-place).
Why reverted
A controlled A/B showed ~2.5% (systems 64.1 vs 66.0 ms) — below the 5–7% floor, and it adds module state. Not a clear win ⇒ discard (consistent with iter 8).

Left on the table

The remaining 1 h-catch-up cost is dominated by dispatchPhase (routes, ~63% inclusive). These are bigger but blocked by behaviour-preservation or invasiveness — a correct optimizer outcome to report, not force.

  • Per-step weather sampling (routeCapacityMultiplierForWeather → weather, ~30% incl): recomputed every 1 s step over weather that only changes per dwell (30 min–4 h). A per-dwell cache is the win but is behaviour-risky (intricate; the golden's single cadence could pass while a latent divergence hides).
  • Per-route-per-step skill multipliers (the iter-3 target): the safe form is hoisting precomputed per-source multipliers into the systems loop, but the calls are spread across routes/drones/orbital (11 sites) — invasive for this budget.
  • Route cargo planning (planRouteCargo / destinationHeadroom, ~24% / ~16% incl): per-route-per-step headroom math; needs analysis to separate genuinely-dynamic from static-per-gap work.
  • Economy advance (~4.9 ms, the bulk of the 1 s push): already heavily memoized (derivationsSignature / getDerivationsMemo); the comments explicitly reject the obvious shortcuts (number-hash keys). Near the noise floor — further wins need a different angle.
  • Note: the 24 h gap (~50 s/sample at baseline) was excluded from the per-iteration gate (too slow); it benefits identically from iters 1–2 and is opt-in via --include-24h.
