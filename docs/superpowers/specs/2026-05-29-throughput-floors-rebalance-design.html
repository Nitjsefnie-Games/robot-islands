Production rates from real throughput, and a floor-upgrade system

Context Follows the SI mass-balance rework (recipes kg-balanced) and the energy SI rebalance (power in kW). This pass grounds time: recompute all 249 recipe cycleSec and normalize extraction amounts from real 24/7 nameplate throughput, area-normalized to our 1 m²/tile buildings — and add a universal floor-upgrade mechanic so players scale per-tile throughput through play.
throughput = density[kg·s⁻¹·m⁻²] × footprint_m² × M × floorMul × skillMul cycleSec(base) = output_kg_per_cycle / (density × footprint_m² × M) output_kg = Σ(output_units × RESOURCE_META.massPerUnitKg) // total mass, all outputs M = 1.4535×10⁻³ (single global; fixes a 1-floor Mine at 20 s: 1/(8.6 × 4 × 20)) footprint: single=1, 2×2=4, 2×3=6, 3×3=9, 4×4=16 m² (1 tile = 1×1 m)
01

Why no math compression

Real areal densities span ~6 orders (water pump ~42 → wafer fab 6.4×10⁻⁵). A single multiplier can only slide that spread, not compress it — so the base rate stays pure real, and the playable dynamic range comes from play: floors ×(1+L) ≤×10, all-skills speed ×10 (already in skilltree.ts via multiplicative recipeRateMul + extractor yield bonuses), and building count — ×100+ per the player, plus offline catch-up. Base = the naive 1-floor / no-skill / 1-building rate.

02

Floor-upgrade mechanic (universal)

Per-building integer floor level L ∈ [0,9] (1–10 floors; cap 10). The physical story behind M: our buildings are effectively multi-story.

aspectrule
Effect (throughput / power output / storage cap / pad slots)× (1 + L) → ≤ ×10
Consumer power draw× (1 + 0.5·L) → ≤ ×5.5
Material cost of L-th upgrade0.8 × base placementCost
Build time of L-th upgradebase build time × (L + 1)
Applies toevery building (producers, consumers, generators, pads, storage)
Skillsmultiplicative with existing rate/yield skills (×10 all-in)

×10 output for ×5.5 power is the efficiency reward for building up. Touches persistence (per-building floorLevel schema bump), economy.ts (scale throughput + power), placement/construction (upgrade cost + progressive time), storage/power-output scaling, and UI (per-building upgrade control).

Phasing

Phase 1
Density→cycleSec generator + extraction & core chain
Reproducible generator (density × footprint × M → cycleSec from recipe outputs + RESOURCE_META + footprints + archetype/density map). Apply to extraction (42) + smelting (21) + the T1–T2 core chain; normalize extraction amounts. Throughput-sanity tests; fix broken cycleSec fixtures.
Phase 2
Remaining recipe cycleSec
Chemistry, manufacturing, electronics, power-fuel, and fantasy/endgame (tier-anchored). Complete all 249; suite green.
Phase 3 — final
Floor-upgrade system
Persistence schema bump (floorLevel), economy throughput/power scaling ×(1+L) / ×(1+0.5L), placement upgrade cost 0.8×, progressive build time ×(L+1), capacity/output scaling, UI upgrade control, multiplicative skill stacking. Tests + in-browser verify.
05

Verification & risks

  • cycleSec is generator-reproducible (a test re-derives a sample); no recipe outside its band; multi-output balances to total-mass throughput.
  • Floor mechanic: ×(1+L) effect, ×(1+0.5L) power, 0.8×/level cost, ×(L+1)/level time, cap 10; floorLevel round-trips persistence; UI works; skills stack multiplicatively.
  • npm test green + tsc + build per phase; browser screenshot of an upgraded building.
  • Risks: M re-tune (cheap — regenerate); fantasy anchoring is abstracted; large fixture churn (never change economy logic to pass a test); this supersedes the deferred coal-EROI retune (mine/coal rates now physics-derived — verify EROI sane); tutorial pacing hints may shift (reconcile on tutorial resume).
