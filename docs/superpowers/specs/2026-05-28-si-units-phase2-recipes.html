Problem

Today's recipes carry the v3-audit's stoichiometry gaps: 1:1 smelter collapse, missing slag in EAF, mass-positive chlor-alkali, ex-nihilo air_separator, 1:N cohort recipes hiding 100× mass errors. Without Phase 2, Phase 1's schema is data with no consumers and Phase 6's CO₂ pool has no inputs.

What's broken today (verified by direct recipe read):

  • smelter at recipes.ts:1363: 1 iron_ore + 1 coal → 1 iron_ingot — no slag, no CO, mass-collapsing.
  • steel_mill at recipes.ts:1430: 1 pig_iron → 1 steel + 1 slag — slag ratio 100% (real BOF: 10-15%).
  • blast_furnace at recipes.ts:1424: 1 iron_ingot + 1 coke → 1 pig_iron — ingot-as-charge shortcut, no flux, no CO₂.
  • electric_arc_furnace: zero slag, zero off-gas. Real EAF: 10-20 kg slag/t.
  • chlor_alkali_plant at recipes.ts:2509: 2 saltwater → 1 chlorine + 1 sodium_hydroxide — chemically impossible at this saltwater ratio, no H₂ output.
  • air_separator at recipes.ts:2570: ∅ → 1 nitrogen + 1 oxygen + 1 argon — 100 kg created from nothing.
  • limekiln at recipes.ts:2188: 1 limestone → 1 quicklime — process CO₂ missing.
  • §3.9 cohort (recipes.ts:2546, 2880, 2883, 2889, 2896, 2902, 2909): every recipe is 1 steel → N component, 100×/16×/10×/8× mass-positive against §2.2 unit-mass anchors.

09

exogenousFlow mechanism

Recipes that legitimately violate Σ inputs = Σ outputs (fuel-combustion CO₂, atmospheric intake, electrode graphite, evaporation, light-ends venting) declare the violation via a new exogenousFlow field on the recipe. The mass-balance auditor reads the tag and skips the equality assertion for that recipe.

9.1 Field declaration

// recipes.ts — extend the Recipe interface near line 1138.
export interface Recipe {
  /* …existing fields… */
  /** Per rev-16 §3.8: declared exogenous mass flow that legitimately
   *  breaks Σ inputs = Σ outputs. The string names the flow's source
   *  (e.g. 'fuel-combustion-CO₂', 'atmosphere',
   *  'electrode-graphite-consumed'). The mass-balance auditor
   *  reads this and skips the equality assertion. The CO₂ accrual
   *  integrator special-cases 'fuel-combustion-CO₂' to add
   *  the accompanying exogenousFlowKg to state.co2Kg. */
  readonly exogenousFlow?: string;
  /** Magnitude of the exogenous flow in kg per cycle. Required when
   *  exogenousFlow is set to a CO₂-bearing tag; optional otherwise. */
  readonly exogenousFlowKg?: number;
}

9.2 Canonical exogenousFlow values (rev-16 §3.8)

  • 'fuel-combustion-CO₂' — limekiln (+5 kg), cement_mill (+4 kg). Integrator adds exogenousFlowKg to state.co2Kg.
  • 'atmosphere' — air_separator. Auditor: input is always satisfied; integrator does not consume air from inventory.
  • 'BF-top-gas-trace' — blast_furnace. Auditor: ±3% rounding tolerance; integrator no-op.
  • 'electrode-graphite-consumed' — electric_arc_furnace. Auditor: Δ-tolerated; integrator no-op.
  • 'evaporation-water-vapor' — concrete_plant. Auditor: Δ-tolerated; integrator no-op.
  • 'hydrocracker-light-ends-vented' — diesel_refinery. Auditor: Δ-tolerated; integrator no-op.
  • 'hydrotreatment-H2S-and-light-gas' — kerosene_refinery. Same.
  • 'process-losses + HCl' — lubricant_refinery. Same.

CO₂ accrual integrator

The piecewise integrator at economy.ts:~1600 (advanceIsland's segment loop) gains a hook: when a recipe produces co2 in its outputs OR carries exogenousFlow: 'fuel-combustion-CO₂', the recipe's per-cycle CO₂ mass is added to state.co2Kg proportionally to the segment's recipe progress.

// economy.ts — inside the segment integrator, where output deltas accrue.
// For each producing building this segment:
for (const [outputId, rate] of Object.entries(perBuildingOutputRates)) {
  // …existing inventory accrual…
  if (outputId === 'co2') {
    state.co2Kg += rate * segmentDurationSec;  // kg per s × s = kg
  }
}
// Exogenous-flow CO₂ — recipes that produce CO₂ outside their balance sheet.
for (const b of producingBuildings) {
  const recipe = resolveRecipe(b, state, nowMs);
  if (!recipe) continue;
  if (recipe.exogenousFlow === 'fuel-combustion-CO₂') {
    const cyclesThisSegment = (perBuildingOutputRates[primaryOutput(b)] ?? 0)
                              * segmentDurationSec / recipe.outputs[primaryOutput(b)]!;
    state.co2Kg += (recipe.exogenousFlowKg ?? 0) * cyclesThisSegment;
  }
}

World totalCo2Kg aggregation. world.totalCo2Kg is recomputed lazily by summing all islands' co2Kg on demand (Phase 6 weather reads need the total). Phase 2 does NOT update world.totalCo2Kg each tick — the per-island accumulator is the source of truth; the aggregate is a derived value computed by Phase 6's weather consumer.

Biogenic CO₂ tagging. charcoal_kiln emits CO₂ that is biogenic (net-zero over forest regrowth cycle, per rev-16 §7.2). Phase 2's accrual hook does NOT distinguish biogenic from fossil CO₂ in state.co2Kg — Phase 6's weather coupling can apply a discount if needed via a second per-recipe tag (biogenic: true) that Phase 2 adds optionally on charcoal_kiln + biomass_plant. The discount logic ships in Phase 6.

Mass-balance auditor

New vitest file src/mass-balance.test.ts. Walks every recipe in RECIPES, computes Δ = Σ outputs × mass − Σ inputs × mass, and asserts Δ = 0 unless one of two exemptions applies.

11.1 Auditor logic

describe('mass-balance auditor — every recipe', () => {
  for (const [recipeId, recipe] of Object.entries(RECIPES)) {
    if (!recipe) continue;
    it(`${recipeId} mass-balances`, () => {
      // Speculative recipes (fantasy chemistry) are exempt.
      if (RECIPE_SPECULATIVE[recipeId as RecipeId]) return;

      const inputMass = Object.entries(recipe.inputs).reduce(
        (acc, [r, n]) => acc + (n ?? 0) * RESOURCE_META[r as ResourceId].massPerUnitKg, 0,
      );
      const outputMass = Object.entries(recipe.outputs).reduce(
        (acc, [r, n]) => acc + (n ?? 0) * RESOURCE_META[r as ResourceId].massPerUnitKg, 0,
      );
      const delta = outputMass - inputMass;

      // exogenousFlow tag declares a legitimate Δ. The auditor accepts ±5% tolerance
      // when the tag is set — exact mass on the doc's worked examples isn't
      // always achievable in integer-rounded recipes.
      if (recipe.exogenousFlow) {
        expect(Math.abs(delta), `${recipeId} declared exogenousFlow but Δ > 5% of input`)
          .toBeLessThanOrEqual(inputMass * 0.05 + 0.5);
        return;
      }

      // No exemption — mass must balance exactly (modulo float tolerance).
      expect(Math.abs(delta), `${recipeId} Δ = ${delta}`).toBeLessThan(0.001);
    });
  }
});
