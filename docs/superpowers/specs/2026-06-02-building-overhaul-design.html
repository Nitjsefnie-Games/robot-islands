Building overhaul — cancel construction, build queue, level badge

Context The construction system already supports parallel build slots (base 1, up to 4 via the Robotics skill sub-path) and per-building construction timers, but a placement is hard-blocked when all slots are full, and there is no way to abort a build. This spec adds four related mechanics: cancel-for-full-refund, a FIFO build queue that feeds the existing slots, mirror skill nodes that scale queue capacity, and a persistent floor-level badge on every building.

Decision
A queued build is a PlacedBuilding flagged queued, paid at enqueue, cancellable for a 100% refund.
Representing a queued build as a committed-but-not-ticking building (rather than a separate pending array) reuses the existing overlap/footprint validation, reserves tiles at enqueue, and makes cancel a single uniform operation (remove + refund) for both queued and running builds. Materials are deducted at enqueue — the analog of today's pay-at-placement — so "refund every spent material" applies the same way whether a build is queued or running.
Queue capacity tracks a fixed 1:2 ratio with running slots via mirror skill nodes (one per existing parallel-slot grantor, granting 2× into queue). Cancel is deliberately distinct from §6.7 demolish (30% scrap) and relocate (half-fee): it is a full refund, available only while construction is incomplete.
00

Existing system we build on

The invariants the design must preserve.

  • Construction is per-building: PlacedBuilding.constructionRemainingMs. While > 0, the building does not produce, contribute power, or accrue maintenance; it ticks down each advanceIsland segment and flips operational at 0. (src/construction.ts)
  • Parallel build slots = parallelBuildSlots(state) = 1 + floor(parallelBuildBonus) + (parallelConstruction ? 1 : 0). Today: robotics.notable.parallelFoundries (parallelBuildCapAdd, magnitude 2.0 → +2) and robotics.keystone.parallelConstruction (structural +1). Full tree = 4 running slots.
  • inProgressBuildCount counts constructionRemainingMs > 0. Both placeBuilding and applyUpgrade reject with reason: 'queue-full' when the count reaches the slot cap; materials deduct at commit.
  • Demolish ≈ 30% scrap (§6.7); relocate = half-fee. Cancel is new and distinct (full refund).
  • Persistence at SCHEMA_VERSION 17; AGENTS.md: bump = migrate. Skill-tree budget guard: ≤ 23 nodes & ≤ 2 filler lever-families per sub-path.
01

Queued-build representation

Why a flagged building beats a separate pending array.

A queued build is a PlacedBuilding committed to the map with queued: true. It occupies its tiles immediately (footprint reserved at enqueue, reusing existing overlap validation), carries its full constructionRemainingMs but does not tick while queued, and holds a queueSeq (monotonic per-island enqueue counter — sourced from IslandState, not wall-clock, since the pure layer forbids Date.now()) for deterministic FIFO order.

Separate queue array
  • Duplicates footprint validation off-map.
  • Must re-validate the footprint at promotion (tile may be taken).
  • Cancel has two code paths (queued vs running).
Flagged PlacedBuilding (chosen)
  • Reuses overlap validation; tiles reserved at enqueue.
  • No promotion-time re-validation, no "tile got taken" race.
  • Cancel is one uniform remove+refund for queued and running.

Predicate changes: inProgressBuildCount counts running only (constructionRemainingMs > 0 && !queued); new queuedBuildCount counts queued === true. tickConstruction and nextConstructionCompletionMs skip queued buildings (no decrement, no scheduled completion). computeRates already zeros production for constructionRemainingMs > 0, which correctly covers queued builds — no change there.

03

Materials timing — deduct at enqueue

Enqueue is the analog of today's placement commit.

  • Affordability checked once, at enqueue; inventory deducted immediately (full placement cost incl. terrain-modifier upfront cost per §8.9, or upgrade cost for an upgrade job).
  • Each enqueue sees the post-deduction balance, so the next enqueue gates against the right inventory.
  • No promotion-time "can't afford it now" failure mode to handle.
  • Cancel refunds the stored cost — "refund every spent material" applies uniformly because materials were spent at enqueue.
04

Cancel = 100% refund

One action, two shapes — available whenever construction is incomplete (queued or running).

  • Cancel a fresh placement → remove the building entirely; refund full placement cost (incl. terrain-modifier upfront); strip any storage-cap contribution the placement added; free the slot and trigger promotion (§05). A queued placement uses this same path — it never started ticking.
  • Cancel an in-progress upgrade → building persists; revert to pre-upgrade floor level; clear constructionRemainingMs; refund the upgrade cost only.

New pure function cancelConstruction(spec, state, buildingId): CancelResult in placement.ts, returning the refunded basket (mirrors DemolishResult's shape). Storage-cap mutation ordering follows demolishBuilding's discipline.

05

Promotion (pure economy hook)

FIFO promotion at the construction-completion boundary the integrator already splits on.

When a running build completes inside advanceIsland — at the nextConstructionCompletionMs boundary — and a slot frees, promote the FIFO head (lowest queueSeq): clear its queued flag so it begins ticking that same tick. Promote repeatedly if multiple slots free in one segment, until slots are full or the queue is empty. No separate scheduler; deterministic for both 1-frame and 24h offline-catchup.

06

UI

A draggable queue window (top-left) and a persistent level badge (bottom-right).

  • Build-queue window — windowed, draggable panel reusing mountPanel + makePanelDraggable (the bottom-right HUD's pattern), registered in the top-left zone (Zone.TL, currently unused). Lists running builds (with progress) then queued builds (FIFO), each with a full-refund Cancel button; shows running N/slots and queued M/queueSlots. All handlers dispatch through the input.ts registry (dispatchAction) — no ad-hoc DOM handlers.
  • Enqueue path — when running slots are full but queue has space, the existing 'queue-full' reject becomes an enqueue; only a full queue hard-blocks.
  • Level badge — added to building-alerts-overlay.ts (already imports floorLevel). Persistent, always shown (incl. L1), bottom-right corner. Bottom-left stays free; top corners remain construction (TL) / maintenance (TR). Pure PixiJS Graphics on the existing throttle.

09

Verification

Pure-layer + TDD; renderer not required for logic tests.

Enqueue. Deducts materials; affordability gate; blocks only when queuedBuildCount >= queuedBuildSlots.
Count split. inProgressBuildCount excludes queued; running gate unaffected by queued builds.
Promotion. FIFO order; multiple promotions in one segment; offline-catchup correctness.
Cancel placement. Building removed (running + queued), full refund, storage caps restored, slot freed + promotion fires.
Cancel upgrade. Level reverted, upgrade cost refunded, building persists.
Capacity math. queuedBuildSlots at empty/partial/full tree; 1:2 ratio at empty and full.
Guards. Budget guard passes (Robotics ≤ 23); migration tests (§07) green; npm run build + npm test clean.
