Building-gated tiered routes

Approved — brainstorming

Higher-throughput inter-island routes are currently unreachable — every route is hardcoded T1 cargo. This makes route tier a property of the transport building that hosts it: place a better dock, route from it, get a faster route.

How a route is created

1 · Place a transport building
Cost = the building's existing placementCost. The route itself is free — you pay by placing the building.
↓
2 · FREIGHT GRID form
Pick island → pick a transport building with no route yet → pick destination → commit. The building's def sets the tier.
↓
3 · Route created
{ sourceBuildingId, type, capacityPerSec, transitTimeSec } — type & rate from routeProfileForBuilding(defId).
↓
4 · tickRoutes dispatch
Unchanged — drone/airship/teleporter dispatch & fuel debits already exist; capacity/transit read from the per-route fields.
↓ demolish the source building
↳ Route drains
route.draining = true — in-flight cargo finishes, then tickRoutes prunes the route. No cargo lost.

Core rules

Building-gated
Every route needs its tier's building on the source island — closes the "T1 routes are free" gap too.
One route per building
A building hosts at most one route. Want a second route? Place a second building. Matches SPEC §9.5's Mass Driver rule.
Building owns the route
Route carries sourceBuildingId; type & rate are derived from that building's def at creation.
Teleporter = paired
A teleporter route also needs a Teleporter Pad on the destination island — gate only; the dest pad is not consumed.
Demolish drains
Removing the source building drains its route via the mechanism shipped in 34594a6.
Legacy routes
Saved routes with no sourceBuildingId keep running as plain cargo — grandfathered, just not building-linked.

UI — FREIGHT GRID panel

Creation form (Approach A). FROM becomes two selects: island → transport building (only buildings with no route yet; each option shows tier + rate, e.g. Airship Dock · airship · 2.0 u/s). TO stays an island select; for a teleporter source, commit requires a pad on the destination island.

Active ledger gets an island filter. A dropdown above the ACTIVE list — "All islands" + each populated island by name — filters rows to routes whose source is the selected island. The filter value joins the ledger's structural signature, so changing it rebuilds rows through the existing reconcile path (34594a6).

Edge cases

  • No eligible building / all buildings already host a route — commit disabled, readout explains.
  • Teleporter source, no pad on the destination — commit disabled, readout explains.
  • Building demolished mid-flight — route drains, cargo lands, route is pruned. Nothing lost.
  • Legacy route — runs as before; shown in the ledger; simply not demolish-coupled.
