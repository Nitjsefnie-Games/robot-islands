Design spec · Robot Islands

Map-placement UI for artificial-island construction

Context Construction (§2.5) is a blocking modal where the target position is typed as blind X/Y numbers with no map preview, while the drone panel (and every other HUD panel) is a movable, resizable, persisted window. This spec makes construction a drone-style panel, adds a live draggable ghost ellipse on the map, and introduces a rule that the island footprint must lie entirely in discovered-or-visible space.

Decision
Convert construction to a movable HUD panel and pair it with a world-space ghost ellipse the player drags and resizes.
The blocking modal is what prevents map interaction. Swapping mountModal for mountPanel + window-manager.ts makes the window non-blocking (and draggable/resizable/persisted, matching drones), which is the enabler for a live ghost. The ghost renders in the world container so it scales and pans exactly like islands. Validity shows as ghost color (cyan/red) + reason; the discovery rule is enforced in the pure layer so the server honors it.
Deliberately not: a "drag-out to draw" gesture, collision-blocked dragging, reusing the building-footprint preview, or a DOM/CSS ghost.
01

Before vs. after

Today — blocking modal
  • mountModal / ri-modal — centered, blocking backdrop.
  • Position typed as two blind X/Y number fields (default 100,100).
  • No map preview — no ghost, ring, or click-to-place.
  • Overlap reported as amber text only after typing.
  • No discovery/vision constraint — can target unseen ocean.
After — panel + live ghost
  • mountPanel / ri-panel — drag, resize, persisted, non-blocking.
  • Drag ghost body to move; handles to resize; sliders & X/Y synced.
  • World-space ghost ellipse scales with the camera like islands.
  • Cyan valid / red invalid with a specific reason, live.
  • Footprint must be fully in revealedCells — enforced pure + server.

03

Interaction flow

  1. Press C / click Construct → movable panel opens at its persisted position; a ghost spawns at a default valid spot near the founder island.
  2. Pick founder (T3+ with operational platform_constructor), biome chips, size sliders (max = tier cap 8/12/16). Ghost updates live.
  3. Drag the ghost body to move; drag handles to resize (clamped to cap). Sliders and X/Y fields stay synced both ways.
  4. Ghost is cyan when valid, red on overlap / off-map / extends-into-unknown; the panel status line gives the reason.
  5. Cost grid shows have / need; Construct enables only when valid and affordable.
  6. Construct → gateway (LOCAL direct / REMOTE intent) → server re-validates incl. discovery → island inserted, rebuildWorldLayers(). On a successful construct the panel closes (the ghost clears); reopen it (key C) to place another island.
