03

Correctness findings

This is fundamentally a correctness rewrite — the tutorial must match real tiles, costs, recipes, and reachability.

The "freshwater" bug (representative)

Phase-7 step 02 says "freshwater cluster." There is no freshwater tile — water_wheel.requiredTile = ['water']. The home water cluster is tile-type 'water'; comments mislabel it "fresh-water" (a fresh_water resource exists, from a Well, but no such tile). All texts must use real tile names.

Legacy gap: Smelter needs clay, no clay step

smelter costs stone 400 / clay 100 / wood 20; starter has 0 clay and the legacy chain had no clay step. → insert a Clay Pit Extractor step (clay_pit_extractor, stone 140 / wood 80, tile clay_pit) before the Smelter.

05

Starting-inventory reachability

Starter (rev-9): stone 1200, wood 600, iron_ore 30, coal 80, iron_ingot 60, bolt 25, limestone 15, saltwater_cell 4, foundation_kit 1.

Verdict: just-sufficient to start, but power-bound — not material-bound.

  • iron_ingot (60) is the tightest material, but the iron-replenishers — Mine, Clay Pit, Smelter — cost 0 iron_ingot, so iron production is always bootstrappable. No iron softlock.
  • First settlement is free — 1 foundation_kit in the starter (no 200-bolt Kit Assembler needed to start).
  • Power is the binding constraint. Supply: Windmill 15, Water Wheel 20. Draw: Mine 25, Quarry 30, Clay Pit 30, Smelter 50, Workshop 60. One source can't fully run even one Mine; the full early set ≈ 200 ≈ 13 windmills ≈ 1000+ wood vs 600 starter. Saved by brownout (factor = min(1, produced/consumed), throttle not stop) and the 0-power Logger (free wood) — the ramp climbs out.
  • Player starts with zero power buildings → first build must be power (confirms first-steps ordering). This is the deferred rev-17 reachability gate.
