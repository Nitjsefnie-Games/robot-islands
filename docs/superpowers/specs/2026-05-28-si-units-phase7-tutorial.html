01

Problem

After six phases of mechanical realism work, the game is materially different from what the existing tutorial teaches. A new player following the old tutorial will hit a wall when they try to build a Cell Press without copper, a Blast Furnace without limestone, or a Wind Turbine before the steel chain is online.

What the old tutorial assumed: Mine + Workshop + Wind Turbine + Solar; pre-SI placement costs; binary heat gates; trapezoidal solar; no CO₂; no map picker; no real-chemistry chain. None of this survives the rework.

What the new tutorial must cover: 25+ distinct mechanics, in an order that doesn't break the chain-dependency graph. Per rev-16 §11.2: building placement → tile gates → cycleSec → power balance → adjacency → heat budget → copper prospecting → limestone for steel → construction time → storage caps → maintenance → day-night solar (with chosen location) → battery bootstrap (rev-9 saltwater_cell chain) → drones → discovery → settlement → foundation kits → antennas → cargo routes → skill tree → CO₂ tracker → tier transitions → reactor toxicity → biome gating → weather → land reclamation → orbital → network consciousness → tier reset → lattice → genesis milestone.

Why the playtest is mandatory. Trigger conditions fire on game events; an incorrect predicate manifests as "hint never appears" or "hint appears prematurely." Static testing can verify the predicate function; only a real play session verifies the FEEL — that the hint arrives in time to help, that consecutive hints don't queue up in a wall of text, that the player isn't blocked by a missing prerequisite.

05

tutorial.ts state machine

New module. Step definitions, trigger polling, dismissal logic. Pure (no DOM) — the UI layer consumes the state.

// tutorial.ts
export interface TutorialStep {
  readonly id: string;
  readonly triggerCondition: (world: WorldSnapshot) => boolean;
  readonly hint: string;
  readonly expectedAction: string | null;
  readonly dismissalCondition: (world: WorldSnapshot) => boolean;
  readonly priority: 'critical' | 'recommended' | 'optional';
}

export const TUTORIAL_STEPS: ReadonlyArray<TutorialStep> = [
  {
    id: '01_map_picker',
    triggerCondition: (w) => w.playerLat == null,
    hint: 'Choose your location — real sunrise will follow real time at that spot.',
    expectedAction: 'open_map_picker',
    dismissalCondition: (w) => w.playerLat != null,
    priority: 'critical',
  },
  // …31 more entries…
];

export function currentStep(world: WorldSnapshot): TutorialStep | null {
  const completed = new Set(world.tutorialState?.completed ?? []);
  for (const step of TUTORIAL_STEPS) {
    if (completed.has(step.id)) continue;
    if (step.triggerCondition(world)) return step;
  }
  return null;
}

export function checkDismissals(world: WorldSnapshot): string[] {
  const completed = new Set(world.tutorialState?.completed ?? []);
  const newlyCompleted: string[] = [];
  for (const step of TUTORIAL_STEPS) {
    if (completed.has(step.id)) continue;
    if (step.dismissalCondition(world)) newlyCompleted.push(step.id);
  }
  return newlyCompleted;
}

Per-tick polling. main.ts's ticker calls currentStep(world) + checkDismissals(world) once per frame (or once per game-second; the polling frequency is implementer's choice — frame-rate is fine since predicates are cheap). Dismissals append to world.tutorialState.completed; new triggers update the active hint.

5.1 Save/load — tutorial completion

tutorialState is already declared in SerializedWorld at persistence.ts:145. Phase 7 doesn't add a new schema field — only uses the existing one. Pre-Phase-7 saves load with tutorialState: undefined; the currentStep helper treats it as empty completed list (no steps completed); the player sees the tutorial from step 1 on next session. Mitigation: if a player wants to skip the tutorial entirely, the settings panel gains a "Mark tutorial complete" action that fills completed with all 32 ids.

06

tutorial-ui.ts hint overlay

DOM-based hint surface. Floats over the canvas in the top-right corner. Dismisses automatically when the step's dismissal condition triggers; player can also click to dismiss manually.

// tutorial-ui.ts
let hintEl: HTMLElement | null = null;

export function refreshTutorialHint(world: WorldSnapshot): void {
  const step = currentStep(world);
  if (!step) {
    if (hintEl) {
      hintEl.remove();
      hintEl = null;
    }
    return;
  }
  if (!hintEl) {
    hintEl = document.createElement('div');
    hintEl.className = 'tutorial-hint';
    document.body.appendChild(hintEl);
    hintEl.addEventListener('click', () => {
      // Manual dismiss — mark current step completed.
      world.tutorialState = world.tutorialState ?? { completed: [], current: null };
      world.tutorialState.completed.push(step.id);
      refreshTutorialHint(world);
    });
  }
  hintEl.innerHTML = `
    <div class="tut-num">Step ${step.id}</div>
    <div class="tut-text">${step.hint}</div>
    <div class="tut-action">${step.expectedAction ?? ''}</div>
  `;
}

6.1 Styling

Hint overlay uses the existing ui.css palette. Top-right anchored; fades in/out; max-width 320 px. Click-through behaviour: clicking on the hint dismisses it but doesn't pass through to the canvas beneath (the player still sees their map).

6.2 Settings panel — "Skip Tutorial" action

Per spec §05.1, the settings panel gains a "Mark tutorial complete" button that fills completed with all 32 step ids. The player can also re-open the tutorial via a "Restart Tutorial" action that clears completed and resumes from step 1.

Test plan

For each of 32 steps: construct a world state satisfying the trigger; assert currentStep returns that step. Then mutate state to satisfy dismissal; assert checkDismissals returns the step's id.

9.2 Reachability invariant (binding gate)

Per spec §07. Must pass.

9.3 Step ordering

A simulated playthrough through simulateOptimalPath exposes steps as they fire. Assert step ids appear in the expected order (1-32) — no step fires before its prerequisite step has dismissed.

9.4 Save migration with mid-tutorial state

A v15 save with tutorialState.completed = ['01_map_picker', '02_bootstrap_power'] loads correctly; currentStep returns step 3 (next in sequence). Skipped pre-Phase-7 saves with tutorialState: undefined load with empty completed.

10

Human playtest checklist (8 h)

Manual gate. The subagent landed code is green per the unit tests; the playtest verifies the FEEL.

Playtest cannot be skipped
Per rev-16 §11.4 — tutorial UX trigger timing, hint placement, and player-skip behaviour cannot be subagent-verified. The 8 h budget is a hard floor; expect to log 2-4 fix commits as the playtest surfaces issues.

10.1 Fresh-new-game playthrough

  • New world; verify map picker fires on session start.
  • Pick a temperate-latitude location (e.g. 40° N, NYC).
  • Walk steps 1-13 to first battery_bank. Time the run; should be ≤ 45 min.
  • Verify each hint fires at the right moment (after the trigger condition becomes true, before the player gets stuck).
  • Verify each hint dismisses at the right moment (after the expected action completes).

10.2 High-latitude playthrough

  • Restart tutorial; pick 64° N (Reykjavik). Solar near-zero in winter.
  • Verify the player can still reach battery_bank via biomass + coal — the tutorial doesn't break.
  • Document any issues with "low solar production" not being surfaced clearly enough.

10.3 Skip / restart flow

  • From any mid-tutorial state, hit "Skip Tutorial" — hint disappears, no more hints fire.
  • Hit "Restart Tutorial" — step 1 reappears, completed list resets.

10.4 Late-game tutorial steps

  • Steps 22-32 cover endgame mechanics. Verify they fire at the right player progress points.
  • Step 32 (Genesis Cell) only fires once; subsequent crafts don't re-trigger.

10.5 Pre-Phase-7 save load

  • Load an existing live save (the user's 27.5 MB /tmp/robot-islands-save.json) post-Phase-7 deploy. Verify the tutorial picks up where it can — the player has already built batteries, so steps 1-14 should dismiss en-masse on first poll; steps 15+ fire as their conditions become true.

10.6 Fix-commit budget

Expect ~2-4 fix commits during the playtest, each addressing a specific issue (hint timing, missed trigger, wrong order). Per rev-16 §11.4 the 8h budget includes the time for these fixes.

13

Non-goals

Tutorial localization
Hint strings are English-only. Future internationalization work can extract them into a strings file.
Cinematic-style "tutorial mission"
Phase 7 ships hints + nudges, not a scripted tutorial level. The player plays the real game with guidance overlays.
Per-step animated highlighting
Step "place a Mine on an ore tile" could highlight target tiles. Phase 7 ships text hints only; future content can add visual overlays.
Achievements / badges for step completion
Out of scope. The 32-step list is internal to the tutorial; the player doesn't see a "Step 14/32" progress counter.
Branching tutorial paths
All 32 steps fire in a single linear sequence. A player who picks an alternative starting biome (when biome selection ships in some future content) might want a different tutorial — not addressed.
In-tutorial tooltip system
Hint overlay is separate from per-element tooltips. Existing tooltips stay; tutorial adds a parallel hint surface.
