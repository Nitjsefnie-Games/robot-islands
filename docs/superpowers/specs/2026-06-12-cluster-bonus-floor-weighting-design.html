Design spec · Robot Islands · §4.5

Cluster bonus — floor-weighted capacity

Context The §4.5 cluster bonus rewards a raw building head-count, blind to floor level. That makes wide-spreading scale superlinearly while floor-upgrades stay a space-only fallback — the two systems fight, and the bonus is too strong. This spec rebases cluster size on floor-capacity and cuts the rate, so floor-upgraded buildings are rewarded and wide-spam is tamed.

Decision
Cluster size counts neighbors' floor-capacity, not heads — linear, uncapped, at half the rate.
Building i's multiplier becomes 1 + r·(K − c_i) where c_i = 1 + floorLevel_i and K = Σ c_j over the same-category component, with r cut 0.10 → 0.05. Upgrade beats add-wide on material at every cluster size (and costs zero tiles), so floors become the better long-term investment; wide stays viable for raw output when space is free.
Deliberately not capped or curved — the magnitude is controlled by the rate cut alone. A building's own height is excluded from its own bonus, so a lone tall building gets no self-cluster bonus (identical to today).
01

Problem & root cause

Both symptoms — "floors aren't rewarded" and "cluster is too strong" — trace to one line.

In adjacency.ts clusterBonusMuls, cluster size is a raw head-count (compSize += 1 per building, blind to floorLevel), and every member receives 1 + (k − 1) · 0.10.

Floors aren't rewarded: a floor-10 building (×10 output, 1 tile) adds the same +1 to k as a floor-1 shack, and going tall to save space shrinks head-count, shrinking the bonus. Marginal "add a base building vs upgrade one" (Δoutput / Δmaterial) shows adding wide beats upgrading at every cluster size. Cluster is too strong: 1 + (k − 1)·0.10 is uncapped — a k=9 spam-cluster already gives ×1.8 to every member and keeps climbing.

02

Before vs. after

Head-count (today)
  • Cluster size = number of buildings; floors invisible.
  • mul = 1 + (k − 1)·0.10, uniform across the cluster.
  • Adding wide always beats upgrading per material.
  • Uncapped superlinear growth in head-count.
  • Floor upgrades = space-only fallback.
Floor-weighted, neighbors-only (proposed)
  • Cluster size = sum of neighbors' floor-capacity.
  • mul_i = 1 + 0.05·(K − c_i), per-building by floor.
  • Upgrade beats add-wide per material at every k; 0 tiles.
  • Wide tamed by the rate cut; floors carry the scaling.
  • Lone tall building gets no self-bonus (unchanged).
03

The model

Per-building multiplier; same connectivity/component rule as §4.4, only the size→multiplier step changes.

mul_i = 1 + r · (K − c_i)

  c_i = 1 + floorLevel_i        (the building's own capacity == floorEffectMul)
  K   = Σ c_j  over the same-category 4-connected component
  r   = CATEGORY_ADJACENCY_RATE[category]   (0.10 → 0.05)

Applies to recipe rate and generator power OUTPUT (consumption excluded), as today.

Key property — floor-1 baseline is unchanged but for the rate. When every member is floor-1, c_j = 1, so K − c_i = (k − 1) and the formula collapses to the old 1 + (k − 1)·r. The only change at the all-floor-1 baseline is the rate cut; divergence appears strictly once floors > 1 exist.

06

Verification

Floor-1 collapse parity. An all-floor-1 cluster of size k yields 1 + (k − 1)·0.05 for every member.
Per-building self-exclusion. In a mixed cluster, each member's multiplier excludes its own 1 + floorLevel; a taller neighbor raises others' multipliers but not its own.
No self-bonus. A lone building (any floor) → multiplier 1.0.
Generator power path. Per-building cluster mul flows into power output (not consumption) for clustered generators.
Economy/power tests updated. Old 1 + (k−1)·0.10 expected values rebased to 0.05 + floor-weighted results.
Green build. npm run build + vitest run pass.

08

Out of scope

Caps / diminishing curves
Explicitly rejected — linear, uncapped chosen. Magnitude is controlled by the rate alone.
Per-category rate differentiation
All categories stay at 0.05 for now; the per-category table is retained so values can diverge later without code changes.
Upgrade cost / time / power-draw scaling
Unchanged. This spec only rebalances the cluster term.
