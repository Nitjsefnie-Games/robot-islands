Design spec · robot-islands · §11 telemetry overlay

Slight red tint on antennas whose coverage disc is fully covered by other antennas.

Context The antenna overlay (antenna-overlay.ts) draws one cyan ring per Antenna building so the player can see drone-telemetry coverage. As the player accumulates antennas, some end up entirely redundant — their disc sits inside the union of other antennas' discs. There is currently no visual hint of this, so the player cannot tell at a glance which antennas they could safely demolish to reclaim power / footprint. This spec adds a slight red tint to such fully-redundant antennas.

Decision
Approximate the fully-covered-disc test by sampling 24 perimeter points plus the centre against the union of all OTHER antennas' discs, and blend a small amount of rust into the cyan overlay colour when the test passes — exact blend value tuned design-time in §05's slider and hardcoded as a single REDUNDANT_COLOR constant.
Sample-based — not an exact circle-cover proof. Both directions of error (false positive, false negative) are bounded by sample count and are tolerable because this is a visual hint, not a removal-safety guarantee. The player still makes the demolition decision.
Scope of "other antennas" is global — every antenna on every populated island. This matches how computeSignalRanges already flattens islands into one island-agnostic SignalRange[] for the drone-tick consumer in drones.ts; any per-island scoping would diverge from the actual game mechanic.
01

Background — what an antenna is

The minimum readers need to understand why the redundancy predicate is shaped the way it is.

Game role. Antennas are §11 telemetry relays. Drones outside any antenna's signal range silently drop their scanned cells (drones.ts:635, 666, 679). Without an on-map indication the player can't tell which corners of the map have coverage. The cyan overlay is that indication.

Data shape. A SignalRange is just three numbers: cx, cy, radius — no identity, no island binding (antenna.ts:32-36). The flat SignalRange[] is produced by computeSignalRanges(populatedIslands) (antenna.ts:46-63) and consumed island-agnostically. The same shape feeds the overlay and the drone tick.

02

The redundancy predicate

A clean geometric definition, the chosen approximation, and an honest accounting of its error modes.

Exact definition. Antenna A is fully redundant iff its entire closed coverage disc D(A) is contained in the union of the closed coverage discs of all OTHER antennas:

D(A) ⊆ ⋃ { D(B) : B ≠ A, B ∈ allAntennas }

Approximation. Compute exact disc-in-union cover is annoying (involves boundary-arc clipping). We instead sample N = 24 evenly-spaced perimeter points plus the centre (25 points total) and check each point against the union via the existing pointInSignalRange helper. If every sampled point lies inside at least one other disc, we call A redundant.

Why N = 24 + centre. The chord length between adjacent perimeter samples at N = 24 is 2 r sin(π / 24) ≈ 0.26 r — for the smallest tier (radius 80) that's a ~21-tile chord. Uncovered slivers smaller than that may still evade detection (false positive), but the threshold is tight enough that the visual hint matches what the player would judge by eye. Cost is 1.5× N=16's per-frame point tests on a path that fires on overlay rebuild, not every frame — negligible. The centre sample catches the degenerate case where every perimeter point happens to graze a different disc but the interior is uncovered (rare with one other large disc, costless to defend against).

Honest error accounting. Both directions are possible:

  • · False positive (tint applied, antenna actually has an uncovered sliver): a thin uncovered arc between two sample points. The player sees red, demolishes — and discovers a small dead corridor in the previously-covered region.
  • · False negative (no tint, antenna is actually redundant): sampling happens to find an uncovered point that's, in fact, the only such point in a measure-zero set. Player misses a free demolition.

Both are tolerable. The player's mental model is "ring inside other rings = probably safe to remove" — exact cover is not what's promised. The brief's framing of "false positives are not tolerable" is contradicted by any finite-sample method and is rejected here in favour of "bounded error, visual hint only."

Signature. SignalRange has no identity, so the caller filters self out before calling:

export function isAntennaRedundant(
  test: SignalRange,
  others: ReadonlyArray<SignalRange>,
  samples: number = 24,
): boolean

Pure layer — no PixiJS import, no Graphics, no DOM. Lives in src/antenna.ts next to the existing pure helpers (keeps the antenna math in one module; no new file needed).

Algorithm.

function isAntennaRedundant(test, others, samples = 24) {
  if (others.length === 0) return false;
  // Centre point.
  if (!pointInSignalRange(others, test.cx, test.cy)) return false;
  // N evenly-spaced perimeter points.
  for (let k = 0; k < samples; k++) {
    const theta = (2 * Math.PI * k) / samples;
    const x = test.cx + test.radius * Math.cos(theta);
    const y = test.cy + test.radius * Math.sin(theta);
    if (!pointInSignalRange(others, x, y)) return false;
  }
  return true;
}

Re-uses the union test pointInSignalRange (antenna.ts:66-77) — the same predicate the drone tick uses to decide if a cell is in range. Same semantic, no new truth definition.

04

Scope: which antennas count as "other"

Same island, same kind, or every antenna in the world? Pick a defensible answer.

Rejected — per-island scoping
  • Would tint T as redundant only if an antenna on its own island covers it — even when an antenna on a neighbouring island covers it equally well.
  • Diverges from the actual game mechanic: drones.ts:635 uses a single global union; per-island scoping would lie about coverage.
  • Requires re-shaping SignalRange to carry island id (currently doesn't — antenna.ts:32-36).
Chosen — global union of all antennas
  • Mirrors the existing computeSignalRanges(populated) → flat list → pointInSignalRange pipeline already shipping in drones.ts.
  • "Other antennas" means every antenna in the result of computeSignalRanges(world.islands.filter(populated)) with the test antenna sliced out by index.
  • No change to SignalRange shape.

Same kind? Also rejected. Redundancy is a geometric property of discs, not of tier labels. A T1 (radius 80) fully inside a T6 (radius 700) is redundant for telemetry purposes — that's exactly the case the player wants the hint for ("I have a tiny early-game beacon sitting inside my late-game tower").

05

Visual treatment — tune the blend, copy the value

A slight rust blend into the cyan overlay, tuned design-time in the slider below and hardcoded as a single constant.

The overlay currently draws cyan (SIGNAL_COLOR = 0x7dd3e8) at fill α 0.04, stroke α 0.22, stroke width 1 (antenna-overlay.ts:19-25, 47-50). When the antenna is redundant we blend a small amount of rust (0xE08B7F, the project palette's --rust) into that colour and use the blended hex as the stroke + fill for that one antenna — same alphas, same width, only the colour changes. The blend amount is a design-time choice: drag the slider below to find a value that reads as "different without shouting", copy the resulting hex, and paste it as the REDUNDANT_COLOR constant in antenna-overlay.ts. Nothing in-game reads the slider; the value is baked in.

Why blend, not fully swap? A redundant antenna is still an antenna — it should read in the same visual register. A full swap to rust would read as "alert"; a tuned blend reads as "different, but recognisable as antenna coverage." The slider lets you find the threshold of "noticeable but quiet."

Why not red (0xFF0000)? Project palette is dark + muted — see ocean.ts blues, AGENTS.md vision-model section. A screaming red breaks the visual register. Blending toward 0xE08B7F (template-21 --rust) at the existing low alphas reads as warm-leaning cyan — different without shouting.

06

Where the redundancy check lives

Pure / render separation, performance, and call-site shape.

Pure layer. isAntennaRedundant is added to src/antenna.ts alongside SignalRange, computeSignalRanges, and pointInSignalRange. No PixiJS import. Testable from antenna.test.ts without a renderer. This matches the project's pure/render rule (AGENTS.md → Architecture → pure layer).

Call site in the overlay. Inside the existing rebuild() in antenna-overlay.ts, the inner loop becomes:

const ranges = computeSignalRanges(populated);
for (let i = 0; i < ranges.length; i++) {
  const r = ranges[i]!;
  // others = ranges with self sliced out
  const others = ranges.slice(0, i).concat(ranges.slice(i + 1));
  const redundant = isAntennaRedundant(r, others);
  const colour = redundant ? REDUNDANT_COLOR : SIGNAL_COLOR;
  const px = r.cx * TILE_PX;
  const py = r.cy * TILE_PX;
  const radiusPx = r.radius * TILE_PX;
  gfx
    .circle(px, py, radiusPx)
    .fill({ color: colour, alpha: RING_FILL_ALPHA })
    .stroke({ color: colour, width: 1, alpha: RING_STROKE_ALPHA });
}

One new module-level constant next to SIGNAL_COLOR: REDUNDANT_COLOR = 0x..., where the hex is the value copied from §05's tuner. Starting point: 0xa3b1ad at 35% blend; tune to taste before merging.

Performance. The naïve nested loop is O(N · (N · 17)) where N = number of antennas. For N = 50 antennas (well above typical mid-game), that's ~42 500 pointInSignalRange point tests per refresh(). refresh() only fires on world-layer rebuild (post-discovery, post-placement — see main.ts:1941), not every frame. Acceptable; no caching needed. If N ever exceeds ~500 (it won't) the spec defers to bounding-box / spatial-hash optimisation.

Allocation note. The ranges.slice().concat() inside the loop allocates O(N) per iteration. Acceptable on a rebuild path that fires O(handful of times per minute). If profiling later shows it, swap to an index-skipping wrapper around pointInSignalRange that takes a "skip this index" parameter — but don't pre-optimise.

08

Edge cases and chosen behaviour

Each case the predicate needs to handle, and what it does.

CaseBehaviourWhy
Single antenna in the world Not redundant (no others) others.length === 0 short-circuit returns false immediately. The lone antenna keeps the cyan tint.
Two co-located identical antennas BOTH tinted red Each is redundant relative to the other; "tinting both is honest" — they would each be safe to remove if the other stayed. No tie-break. The player resolves the ambiguity at demolish time.
Antenna whose disc extends off the world bounds Geometric check is performed regardless "World bounds" aren't a concept the predicate cares about — sampling and disc-cover are pure geometry. A disc whose perimeter pokes off-map is still subject to the same union test against other discs.
Antenna covered everywhere on its perimeter by exactly one other, but centre uncovered Not redundant The centre sample catches this. Pathological in practice (would require a doughnut-shaped union missing only the centre), but cheap to defend against.
Boundary equality (sample exactly on another disc's perimeter) Counts as inside pointInSignalRange uses ≤ r² (antenna.ts:74) — closed disc. Consistent with the drone-tick semantic.
Cluster of N tightly-packed identical antennas All N tinted red Each one's perimeter samples are covered by the union of the other N−1. Same "tinting all is honest" rationale. The player sees a red cluster, picks any subset to demolish, the remaining ones stop being redundant on the next refresh().
Antenna on an unpopulated (player-built but uninhabited) island Not included in either side of the test computeSignalRanges already filters to populated (antenna.ts:46-50 — "Walks ALL populated islands"). Same filter applies to the redundancy check; the overlay never draws a ring for it in the first place.

11

Out of scope

Considered, deferred — named so the reader knows.

Exact disc-cover proof
A true geometric proof of "this circle is contained in the union of those circles" reduces to arc-clipping on the boundary. Not built — the sampling approximation is fine for a visual hint and we don't promise removal safety.
"Partial redundancy" gradient
Could compute "fraction of perimeter covered" and tint progressively (cyan → rust). Rejected for v1 — binary tint reads more clearly at small overlay alphas.
Tooltip / inspector affordance
Click-to-inspect on a redundant antenna with a "covered by: T6 @ (cx,cy), T5 @ (cx,cy)" listing. Deferred; the tint alone meets the v1 goal of "at-a-glance".
Configurable sample count via dev console
The samples parameter exists but isn't exposed to the player. Could be a debug flag if tuning is needed later.
Per-island scoping toggle
Considered, rejected (see §04 — diverges from the global drone-tick semantic). If a future "comms graph per island" mechanic is introduced, revisit.
Integration with the §14.10 T6/Spaceport comm-range contribution
A T6 on a Spaceport island contributes to groundStationCommRange (orbital.ts:367-368). The redundancy check ignores this — a T6 might still be needed for the spaceport even if its disc is geometrically redundant for drone telemetry. Player education problem, not a predicate problem. Deferred to a possible later "do not demolish — required for orbital" badge.
