Path-drone UI + shared launch-mode preview

Context Two gaps in the drone launch UI surfaced today. (1) The T5 path-drawn drone engine (Drone.waypoints + isPathDrawn branch + Path Drone Foundry building + scan-radius=12) is fully wired in drones.ts + building-defs.ts, but no UI ever sets the waypoints argument when calling dispatchDrone — so the player can't actually trigger path-drawn launches even with the foundry built. (2) The launch reticle (drones-ui.ts:583) renders only a screen-space crosshair at the cursor; there's no line from origin to cursor, no trajectory preview, no distance/fuel readout while aiming. Both fit one preview-render surface, so they ship together.

Decision
Extend the tier dropdown with "T5 Path" + add a single shared world-space preview overlay that serves both single-click and path-draw modes.
The tier dropdown gains a new entry '5-path' visible only when origin meets the SPEC §11.6 gate: tierForLevel(level) ≥ 5 AND aiCoreCrafted AND hasOperationalBuilding(path_drone_foundry). Selecting it switches the ARM button's pointer-event handling: instead of single-click-dispatch, clicks accumulate into a per-arm waypointBuffer; double-click finalizes (drops the doubled point); right-click pops the last waypoint; Escape clears + disarms.
A single new launchPreviewGfx: Graphics overlay in the world container renders both modes. Numeric tier (1-4): line from origin tile centre → cursor tile centre, tinted by reachability. T5 Path: solid polyline through committed waypoints + dashed segment from last waypoint to cursor, tinted green if the next waypoint keeps 2 × pathLength ≤ MAX_FUEL_PER_DRONE × DRONE_T5_EFFICIENCY, red otherwise. The existing crosshair reticle stays as-is on the cursor.
No engine changes. dispatchDrone already accepts waypoints?: ReadonlyArray<{x,y}> and routes via isPathDrawn. Path mode passes the buffered waypoints with fuel = ceil(2 × pathLength / DRONE_T5_EFFICIENCY); numeric mode passes undefined (today's behaviour preserved).

02

SPEC citations

What §11.6 prescribes for the UI surface.

§11.6 T5 Path-Drawn Drone (SPEC.md:1224-1253). Available only on islands with Tier 5 access (level 50, AI core crafted, Path Drone Foundry built). "The player draws a path on the world map as a sequence of waypoints. The path can have arbitrary length up to a fuel-imposed limit. The drone follows the path at T5 speed, scanning a wide corridor along the entire route."

Use cases the spec calls out: systematic area sweep (zigzag), targeted long-range probe (straight line into deep unexplored space), rare-feature hunt (search remote areas). All three benefit from arbitrary-shape paths — supports the click-to-add waypoint model rather than a fixed shape.

Failure modes: fuel exhausted (lost), weather destruction (lost), successful return (full data dump). The UI fuel-gating step (refuse the click if it would commit a doomed path) preserves the player's ability to abort the draw rather than commit a known-failing run — strictly an improvement over the spec's silent failure mode.

05

Render layer — single shared overlay

Both modes route through one new Graphics instance to keep render code in one place.

A new field on DroneUiHandle: readonly launchPreviewLayer: Container. Owns a single child launchPreviewGfx: Graphics. The container is added to the world container at mount time (same place as rangeRingLayer) so it pans/zooms with the camera. Visibility tied to launchMode — hidden when disarmed.

Per-frame redraw (called from the existing drones-ui ticker hook):

function paintLaunchPreview(): void {
  launchPreviewGfx.clear();
  if (!launchMode) return;
  const origin = deps.getOriginSpec();
  const cur = cursorTile;
  if (!cur) return;

  if (selectedTier === '5-path') {
    // Solid polyline through committed waypoints.
    const points = [originTile(origin), ...waypointBuffer];
    drawPolyline(launchPreviewGfx, points, COLOR_OK, /*dashed=*/false);
    // Dashed segment from last anchor to cursor.
    const last = points[points.length - 1]!;
    const wouldExceed = (totalLength + dist(last, cur)) * 2 > maxLaunchFuel * DRONE_T5_EFFICIENCY;
    drawSegment(launchPreviewGfx, last, cur, wouldExceed ? COLOR_BAD : COLOR_OK, /*dashed=*/true);
  } else {
    // Numeric tier — line from origin to cursor.
    const dist01 = dist(originTile(origin), cur);
    const outOfRange = dist01 > maxLaunchRange;
    drawSegment(launchPreviewGfx, originTile(origin), cur, outOfRange ? COLOR_BAD : COLOR_OK, /*dashed=*/false);
  }
}

Colors. Reuse the project palette — COLOR_OK = 0x8FA56E (olive), COLOR_BAD = 0xE08B7F (rust). Both already used elsewhere in the UI (matches the reticle color logic).

Dash pattern. Approx 8px on / 4px off in world-pixel space. Pixi 8 has no native dashed-stroke API, so implement as a small helper that emits per-segment moveTo/lineTo pairs along the segment direction. ~15 lines.

06

Click + keyboard handling

Where each event hooks in, and how main.ts's existing click-disambiguation logic extends.

Click (`mouseup` after a small `mousedown` movement). main.ts already gates "is this a launch click" via droneUi.isLaunchMode(). When true, today it calls droneUi.attemptLaunch(targetTile.x, targetTile.y, nowMs). After this spec, that call branches inside attemptLaunch:

attemptLaunch(tileX, tileY, nowMs) {
  if (selectedTier === '5-path') {
    return addWaypoint(tileX, tileY);  // returns { ok, reason }
  }
  return existingSingleClickDispatch(tileX, tileY, nowMs);
}

Double-click. main.ts detects double-clicks via the standard browser dblclick event on the canvas. New: when fired in launch mode + path selected, call droneUi.finalizePath(nowMs). The double-click implies a synthetic single-click first, which will add a duplicate waypoint at the same tile as the previous click. Resolution (locked): finalizePath drops trailing-duplicate waypoints unconditionally — if the last two buffer entries have identical tx/ty, pop the tail before dispatch. This works regardless of browser event ordering (Chrome / Firefox / Safari differ) and handles the rare legitimate case (player intentionally double-tapped the same tile) the same way: dispatch what they meant.

Right-click. Add a contextmenu listener on the canvas; when in path mode, preventDefault() and call droneUi.popWaypoint(). Otherwise let the browser context menu through.

Escape. Bind to the existing key-dispatch registry in input.ts. When path mode is armed, Esc clears the buffer and disarms; otherwise it falls through to whatever Esc does elsewhere (close modal, etc.).

10

Out of scope

Explicitly deferred — named so the reader knows it was considered, not missed.

Saved / named paths
No "Save this path as X" affordance. Each launch is fresh. Future spec could persist favorites if the systematic-sweep use case needs them.
Mid-flight cancel
Once dispatched, the drone runs to completion (or destruction). No abort button.
Scan-corridor outline around the path
Not in v1 per picker decision. The polyline + reticle is enough; corridor capsule could be added later if players want it.
T6 drone UI
T6 (antimatter) drones exist in the type but aren't spec'd for §11.5 dispatch UI. Separate spec when T6 gameplay lands.
Scanner Sat dark-mode mitigation for path drones (SPEC.md:1633)
Engine + UI work both deferred. The unified buffer-and-flush from the drone-scan rework handles dark-mode telemetry generically.
Mobile / touch path drawing
Desktop-only at v1 (consistent with the rest of the game per drone-scan / graphview audits).
