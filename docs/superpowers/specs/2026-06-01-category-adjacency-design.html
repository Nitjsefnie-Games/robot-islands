Design spec · robot-islands

Universal per-category adjacency

Context The tutorial advertises a universal "+10% same-type cluster bonus," but the code only grants it to three building defs (mine, smelter, workshop), at non-uniform magnitudes, capped, and multiplicative across entries. This spec replaces that per-def buff with a single universal per-category rule: incremental, linear, uncapped, applied to recipe rate and generator output.

Decision
Every building gains 1 + n × rate[category] from its distinct same-category physical neighbours — uncapped, linear, on recipe rate and generation.
Replaces the per-def adjacencyBuffs[] system (buggy, non-universal, capped, multiplicative). One rule, one per-category rate table seeded at a uniform 0.10, tunable later.
Deliberately does not touch: skill-tree exotic-pair boosts (kept, they are unlock rewards), gates (separate mechanic), power consumption (unchanged — buffed consumers run faster at the same wattage), and the cross-island lattice (excluded from the category term).
01

The bug

What the tutorial claims versus what the code actually does.

The buff is per-def data, not a rule. computeBuffStack (adjacency.ts:115) reads each def's adjacencyBuffs[]. Only three defs carry a same_def entry; everything else gets nothing. Within an entry it is already incremental but capped (maxMatches); across entries it is multiplicative. It already multiplies recipe rate, not output (economy.ts:1063) — so the tutorial's "output bonus" wording is wrong too.

03

The rule

One multiplier per building, computed from its physical neighbourhood.

categoryMul = 1 + ( distinct same-category buildings touching the
                    focal's physical 4-neighbour border ) × rate[focal.category]

Same-category match on defs[n.defId].category === defs[focal.defId].category. Neighbours are distinct by id (a multi-tile neighbour crossing several border tiles counts once).

"4-neighbour" is the adjacency rule (cardinal / von-Neumann, not diagonal), NOT a cap of 4 neighbours. A 2×2 footprint exposes 8 border tiles and can be surrounded by 6+ distinct same-category neighbours — all count. Uncapped, raw count: no maxMatches, no perimeter normalisation. Larger footprints expose more border and can earn more — intended; deliberate base geometry is rewarded. The cross-island lattice (§13.3) does not feed this term — physical adjacency only.

04

Rate table & application

A new per-category constant, and exactly where the multiplier lands.

New CATEGORY_ADJACENCY_RATE: Record<BuildingCategory, number> in building-defs.ts, every category seeded at 0.10. Categories whose buildings neither run a recipe nor generate power (storage, logistics, cooling) are harmless no-ops — nothing to scale.

Verification

How the implementer confirms correctness.
