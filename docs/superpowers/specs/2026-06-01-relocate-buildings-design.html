Relocate buildings + floor-aware demolish

Context Placed buildings are currently stuck — re-planning base geometry means a full demolish + rebuild. And demolish refunds only the base placement cost, silently forfeiting all floor-upgrade investment. This spec adds same-island relocation for half a building's total invested cost, and fixes demolish to honour that same total.

Decision
One totalInvestedCost(b, def) helper drives both: relocate fee = 50% of it, demolish refund/scrap apply their fractions to it.
Relocate moves a building to a new valid tile on the same island, charging half its total invested cost, keeping all runtime state (just teleport x/y). The UX is an inspector Move button entering the existing ghost-placement mode.
Deliberately NOT in scope: cross-island moves, re-entering construction on move, drag-and-drop, bulk move.
01

The two gaps

Today
  • No way to move a building — only demolish + rebuild from scratch.
  • Demolish refund/scrap use base placementCost only.
  • A floor-9 building forfeits ~88% of its invested resources on demolish.
After
  • Relocate to any valid same-island tile for 50% of total invested cost.
  • Relocated building keeps level, construction timer, maintenance, routes.
  • Demolish refunds the floor-upgrade investment too (same total basis).

03

Shared cost helper

The linchpin both features depend on — "what this building is worth."

totalInvestedCost(b, def)[r] = base[r] + floorLevel(b) × upgradeCost(def)[r]
    base[r]        = placementCostFor(def)[r]
    upgradeCost[r] = ceil(0.8 × base[r])     // existing helper, placement.ts:107

New pure function in src/placement.ts. floorLevel(b) is the existing buildings.ts:187 helper. Example: a 100-stone def at floor 3 → 100 + 3×ceil(0.8×100) = 100 + 3×80 = 340 stone.

04

Relocate

New pure relocateBuilding(spec, state, id, newX, newY, rotation?) in placement.ts.

  1. Find the building by id; { ok:false, reason:'not-found' } if absent.
  2. Validate the destination via the existing validatePlacement(), extended with an optional ignoreBuildingId so the moving building's own footprint is excluded from the overlap check (enables 1-tile shifts, prevents self-collision). All other checks unchanged — bounds, overlap with other buildings, terrain requiredTile, coastal, biome, tier. So a Mine can only move onto another ore/coal tile.
  3. Fee = floor(0.5 × totalInvestedCost[r]) per resource. Affordability-gate against state.inventory ('insufficient-resources' with missing); deduct on success.
  4. Mutate only x/y (and rotation if rotated). Everything else persists: floorLevel, constructionRemainingMs (operational stays operational; mid-construction keeps its timer), maintenance timers, toxicity, disabled, cargoLabel.
  5. Return { ok:true, charged }.

No work needed for routes (keyed by building id) or storage caps (position-independent — same island, same def, same floor).

05

Demolish fix (floor-aware)

In demolishBuilding() (placement.ts:797), swap base-cost reads for totalInvestedCost(b, def).

QuantityBeforeAfter
Per-resource refundfloor(base[r] / 2)floor(totalInvested[r] / 2)
Scrapfloor(0.3 × Σ base)floor(0.3 × Σ totalInvested)

Clamping, array removal, and storage-cap stripping unchanged. Inspector previews updated to match: previewRefundForBuilding(b) already takes the building; previewScrapForBuilding changes signature from defId to the PlacedBuilding (it needs floorLevel) at its single call site. Net: upgraded buildings recover their floor investment instead of forfeiting it.

06

UI wiring

inspector-ui.ts
Move button + dep. Add onMove(target) to InspectorDeps; a "Move" button in the footer beside Demolish, labelled with the fee (e.g. ✥ MOVE · −170 stone), greyed when unaffordable.
placement-ui.ts
Relocate mode. A variant of the ghost-placement flow parameterized by the building being moved: footprint ghost follows cursor, validity via validatePlacement(..., ignoreBuildingId: building.id), click confirms → relocateBuilding() + rebuild layers, Esc/right-click cancels (no fee).
main.ts
Wire onMove. onMove enters the relocate mode; on confirm, calls the pure function and rebuilds world layers — mirrors the existing onDemolish wiring.

Architecture: extend placement-ui with a relocate mode (chosen) over a separate parallel overlay (duplicates ghost/validation) or a demolish-then-replace hack (would drain routes, change the id, lose runtime state).

07

Testing

Pure-layer (no PixiJS/DOM).

totalInvestedCost. floor 0 = base; floor L = base + L×ceil(0.8×base) per resource; multi-resource defs.
relocate happy path. fee = floor(0.5×total) deducted exactly; only x/y changed; floorLevel/construction/maintenance/disabled preserved; returns { ok, charged }.
relocate rejects. overlap with another building; destination fails terrain requiredTile; out of bounds; insufficient-resources (inventory unchanged); not-found.
relocate allows self-overlap. a 1-tile shift onto the building's own current footprint passes via ignoreBuildingId.
validatePlacement ignoreBuildingId. excludes that building's tiles from the occupied set; still rejects overlap with others.
demolish floor-awareness. refund + scrap scale with floorLevel (floor-0 matches today; floor-N strictly larger), clamped to caps.
08

Out of scope

Cross-island relocation
Terrain/economy/networking blast radius; same-island only for v1.
Re-entering construction on move
Rejected — "just teleport"; the building stays operational.
Drag-and-drop UX & bulk move
Ghost-mode button only; one building at a time.
