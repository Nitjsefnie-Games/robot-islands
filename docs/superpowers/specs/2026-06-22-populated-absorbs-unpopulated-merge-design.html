Design spec · robot-islands · SPEC §3.6

Populated islands absorb non-populated neighbors on contact

Context A populated island can grow via the Land Reclamation Hub until its tile footprint touches a neighbor — but the merge engine considers only populated islands, so a discovered-or-undiscovered unsettled neighbor is invisible to it and the growing footprint can overlap an unsettled island with nothing happening. This spec decides that such contact triggers a merge: the populated island absorbs the neighbor.

Decision
A merge fires when two footprints touch and at least one island is populated; the populated island is always the absorber.
The non-populated neighbor is folded in as a new constituent lobe, contributing exactly what it has — land area, biome, terrain seed, and ownership-ledger lobe — and nothing else. Two unpopulated islands never merge. The existing populated-vs-populated "larger absorbs smaller" rule is unchanged; it governs only that case.
Eligibility is geometric contact, independent of the discovered flag. In practice the neighbor is essentially always already discovered before contact (a populated island's vision halo extends 10 tiles past its ellipse, well beyond its footprint), but skipping the flag check keeps the rule simple.
01

Background

Why the current behavior is a gap, and why the fix is small.

findNextMerge (src/island-merge.ts:381) filters to world.islands.filter(s => s.populated) before scanning for overlaps, so an unsettled neighbor is never a merge candidate. §3.6 silently assumes both parties are populated and never states the precondition.

The fix is contained because performMerge (:121) is already state-safe: every step that reads the absorbed island's IslandState — inventory transfer (:238), skill refund (:253), storage credit (:224) — is guarded by absorbedState &&, and an unpopulated island has no buildings/routes/drones to migrate. The settlement-vehicle interplay is also already handled: a settler in transit to the absorbed island is retargeted to the absorber (step 7, :292) and consumed cleanly on arrival at the populated absorber (tickVehicles, src/settlement.ts:896).

02

Before vs after

Behavior when a populated island grows to touch an unsettled neighbor.

Before
  • Only populated islands are merge candidates.
  • A growing footprint overlaps an unsettled neighbor — nothing happens.
  • §3.6 leaves the populated/unpopulated case undefined.
After
  • Contact with any island merges, if ≥1 party is populated.
  • The populated island absorbs the neighbor as a new lobe (biome + terrain preserved).
  • Two unpopulated islands still never merge.
03

Rules

The precondition and absorber-selection, stated explicitly.

  • A merge requires ≥1 populated island; two unpopulated islands never merge.
  • Both populated: larger absorbs smaller, level tiebreak — unchanged (chooseMergeAbsorber).
  • Exactly one populated: the populated island is the absorber regardless of relative size.
  • The absorbed non-populated neighbor transfers only its land / biome / terrain-seed / ownership-ledger lobe; it has no inventory, skill points, level, buildings, or routes to carry.
  • Eligibility is geometric contact, independent of the discovered flag.
  • Merging stays permanent (§3.6 "Joining is permanent").

Change inventory

Widen the candidate scan. Scan pairs where ≥1 island is populated (iterate populated × world.islands, dedupe each pair) so cost stays O(P·N), not O(N²). Cover every participant. Include each participating island's id + ellipse geometry + a populated bit, so a stale null isn't served after an unsettled target appears/changes. Only null results are cached; non-null results are consumed immediately, preserving correctness.

No behavioral change; fix stale comments. Update the comments asserting "both islands are populated" to reflect that the absorbed island may be stateless.

State the precondition. ≥1 populated required; both-populated keeps "larger absorbs smaller"; one-populated → populated absorbs regardless of size, transferring only land/biome/terrain-seed/ledger; eligibility is geometric, not gated on discovered.
