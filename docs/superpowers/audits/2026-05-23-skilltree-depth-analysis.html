01

Evidence — what the catalog actually contains

File:line citations to the source of truth. Every claim below is verifiable in the repo.

The SkillEffect union is broad — the catalog usage is narrow. skilltree.ts:70-137 declares 26 effect kinds, including unlockRecipe, exoticAdjacency, biomeBypass, structural, and placeholder. The header comment (skilltree.ts:20-22) admits the last two are "forward-compat for future sub-paths but no catalog node currently uses them." A grep across the entire file shows the only kind: 'structural' usage is in an unreachable default branch of makeOrbitalNodes's switch (skilltree.ts:500) — every declared orbital sub-path hits an explicit case.

Branch lock is order-only. canSpend at skilltree.ts:641-685 enforces: if a SIBLING sub-path in the same branch has spent ≥ 3 points and isn't complete, the purchase is blocked. The lock lifts the moment that sibling is fully completed — which is reachable by every island given uncapped levels and the geometric skill-point grant floor(1.1^level) (skilltree.ts:298-300). The two committed sub-paths across a branch are sequential, not mutually exclusive.

Tier gates do not gate alternate playstyles — they gate depth on the same axis. tierRequiredForDepth at skilltree.ts:273-279 maps depth → tier (depth 1-2 = T2, 3 = T3, 4 = T4, 5-7 = T5, 8+ = T6). All 15 sub-paths follow the same shape: 1 node at T2, 1 at T2, 1 at T3, 1 at T4, 3 at T5, 8 at T6. Depth in this game is "later you can buy more of the same multiplier" — it's not a tree branching into divergent leaves.

Effect aggregation is purely multiplicative across one axis. effectiveSkillMultipliers at skilltree.ts:829-997 folds every owned node into one bundle. Multiple nodes targeting the same kind compose multiplicatively (e.g. mining.1 (1.05) × mining.4 (1.40) = 1.47×); but no node interacts with another node. There are no synergies — the union of two sub-paths produces no effect that neither produces alone.

Specialization roles are flat penalties + flat buffs. effectiveSpecializationMultipliers at specialization.ts:209-261 implements 5 roles, each a (buffCategory × 1.5 / penaltyOnRest × 0.75) pattern. Declaring a role does not modify which skill-tree nodes are visible, which are reachable, or what their effects do — it is a separate, additive multiplier-layer. Role + skill tree are orthogonal scaling knobs, not interlocking systems.

There is no per-node respec. The only path to unbuying a node is the wholesale executeTierReset at tier-reset.ts:133-150, which clears everything (level, XP, role, sub-path progress, all unlocked nodes) and refunds points. The "you can do everything anyway given enough levels" property combined with no-respec means a poor early commit doesn't even cost you the late-game variety — you just take a longer path through the same total node set.

07

Depth-improvement directions

Eight pickers. Each is a design knob that, if turned, deepens the tree without throwing away existing 225 nodes. Select your preferred option per knob; the panel below assembles them into a copy-able decision packet.

Picker 1
Make more sub-paths mutually exclusive within a branch
Currently the branch lock is order-only — given enough levels, every sub-path is reached. Stronger commitment = true playstyle divergence.
Picker 2
Cross-path synergies (pairs of nodes that gain a bonus when both owned)
Today, nodes are independent additives. Pair-bonuses create genuine combinatorial design space.
Picker 3
Role-specific sub-trees instead of flat multipliers
Today, declaring Foundry just adds a global ×1.5 / ×0.75 layer. The role doesn't reshape the tree at all.
Picker 4
Per-node upgrade levels (vs. binary owned/unowned)
A node could itself hold multiple investment levels with diminishing returns — more granular feedback for SP spent.
Picker 5
Endgame node density
T5/T6 today is 165 of 225 nodes but every one is "more of the same axis." Endgame deserves structurally different content.
Picker 6
Active skills vs. all-passive multipliers
Currently 100% passive. A cooldown-gated active skill adds tactical depth — choosing WHEN, not just WHETHER.
Picker 7
Respec mechanism (currently: only wholesale Tier Reset)
Per-node respec mediates the tension between "commitment matters" and "don't trap players on a bad early click."
Picker 8
Tree visualisation overhaul
Today's UI (skilltree-ui.ts) is 4 columns of text rows. A real tree visual surfaces depth choices the catalog already encodes.
