Phase 1 — Layer 1: adjacency cache

Smallest possible diff that delivers the Layer-1 adjacency speedup. One module-level WeakMap, one cache check.

Pseudocode

// Near skilltree.ts:425 — beside DEFAULT_CATALOG.
const _adjCache = new WeakMap<Graph, Map<NodeId, NodeId[]>>();

// Replace skilltree.ts:1154-1170.
function buildAdjacency(graph: Graph): Map<NodeId, NodeId[]> {
  const cached = _adjCache.get(graph);
  if (cached !== undefined) return cached;
  const adj = new Map<NodeId, NodeId[]>();
  for (const e of graph.edges) {
    if (e.mode === 'and') continue;
    const from = e.from as NodeId;
    const to = e.to as NodeId;
    const fList = adj.get(from) ?? [];
    fList.push(to);
    adj.set(from, fList);
    const tList = adj.get(to) ?? [];
    tList.push(from);
    adj.set(to, tList);
  }
  _adjCache.set(graph, adj);
  return adj;
}

  2. Replace the body of buildAdjacency with the cache-checked version above.
  3. Run npm test. The full suite must pass — this is a pure refactor and no test should change behaviour.

Pseudocode

function computeAuraAmplifiers(state: IslandState, graph: Graph): Map<NodeId, number> {
  // Layer 2 hit — gated on DEFAULT_GRAPH because the version
  // counter tracks state mutation, not graph identity. UI paths
  // with crystal-bound mini-trees pass a different `graph` and
  // must bypass the cache (Layer 1 still helps them).
  if (
    graph === DEFAULT_GRAPH
    && state.auraAmpCache !== null
    && state.auraAmpCacheVersion === state.auraAmpVersion
  ) {
    return state.auraAmpCache;
  }
  const amp = new Map<NodeId, number>();
  const neighbours = buildAdjacency(graph);
  const byId = graphById(graph);   // from Phase 2
  // …existing aura-walk body unchanged (lines 1139-1150)…
  if (graph === DEFAULT_GRAPH) {
    state.auraAmpCache = amp;
    state.auraAmpCacheVersion = state.auraAmpVersion;
  }
  return amp;
}

Binding gate
Home-island mean tick must drop to < 1.8 ms (a 1.5× win from the current ~2.67 ms baseline). If this threshold isn't met, the implementation is broken (either Layer 2 isn't actually hitting on the hot path, or a version bump is missing and the cache is constantly invalidating). Roll back to Phase 0 baseline and re-diagnose with a fresh CPU profile.

If the gate fails

  • Cache always misses: add temporary logging around the auraAmpCacheVersion === auraAmpVersion compare. If auraAmpVersion keeps growing faster than expected, an unintended mutation site is bumping it — usually a forgotten ++ inside a loop instead of after.
  • Cache always hits but tick time didn't drop: confirm graph === DEFAULT_GRAPH evaluates true on the hot path — re-read economy.ts:1600 / 1676 to confirm effectiveSkillMultipliers(state) is called without an explicit graph arg.
  • buildAdjacency still ~25% self: the WeakMap isn't holding the cache entry. Confirm DEFAULT_GRAPH is the same identity at both call sites (it's a const so this should be impossible without a regression elsewhere).

Implementation notes

Useful context for the implementer that didn't rise to the level of a spec requirement.

  • Order matters. Phase 1 + 2 are pure refactors and can land before Phase 3; doing them first means the Layer-1 win is visible even if Phase 4 is delayed.
  • Phase 3 is the single largest commit. It touches 4 files and adds one bump per logical mutation site. Resist the urge to split — the bump sites are useless without the fields, and the fields are useless without the bumps. Land together.
  • The 6 logical bump sites cluster the 9 raw lines. Be careful in refundAndClearMiniTree (skilltree.ts:650-690) — the function has two separate for…of loops (one over nodes, one over edges). A single bump at the function tail is sufficient because they execute together; a bump inside each loop is harmless but redundant.
  • Hot-path graph identity. Verified by re-reading economy.ts:1600 and 1676: both call effectiveSkillMultipliers(state) with no second arg, defaulting to DEFAULT_GRAPH per the signature at skilltree.ts:905. The graph === DEFAULT_GRAPH guard in Phase 4 reliably hits.
  • UI graph identity. skilltree-graphview.ts uses effectiveGraph(state) which returns the same DEFAULT_GRAPH reference when there are zero socket bindings (skilltree.ts:545); only the crystal-bound case allocates a fresh object. So UI paths on a fresh save also benefit from the Layer 2 cache; only the rarer "crystal bound" UI render bypasses it.
  • Don't export the helpers. Keep _adjCache, _byIdCache, graphById module-private. Phase 5 Test 3 uses the behavioural equivalence path; no test imports them directly.
