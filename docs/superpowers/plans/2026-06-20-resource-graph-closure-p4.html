02

Closure decision flow

How each orphan is routed to a mechanism, and which gate proves it safe.

Bootstrap-reachable chains (byproduct loops, placement-cost ordering) are gated by the faithful planner — multi-producer-correct. Non-bootstrap chains (ocean, mid-tier) are gated by the recipe-closure screen. A screen flag means "verify an acyclic producer exists / the consumer is off the producer's critical path," not an automatic reject.

04

Contract & validation

The terminal-tag contract, the per-orphan edit shape, and the exact gate commands. 220 of ~240 resources are 1 kg/unit, so mass-balance is usually integer unit-count balance.

src/recipes.ts — the contract (type + gate)
// RESOURCE_META.terminal — the taxonomy gated by recipes.test.ts
terminal: 'consumed' | `expansion-hook:${string}` | 'gameplay-sink';

// gate: a PRODUCED 'consumed' resource MUST have ≥1 recipe INPUT consumer.
// 'gameplay-sink' (placement/fuel) is NOT gate-checked.
// mass-balance.test.ts: Σ(in.n·massPerUnitKg) === Σ(out.n·massPerUnitKg),
//   unless the RecipeId is in RECIPE_SPECULATIVE ('fantasy chemistry').
Per-orphan edit shapes
// (a) tag-only — consumer already exists
heavy_oil: { massPerUnitKg: 1, volumePerUnitL: 1, terminal: 'consumed' },
ceramic_insulator: { /* mass */ terminal: 'gameplay-sink' },

// (b) recipe consumer — new refine recipe + retag
tar_refinery: { inputs: { wood_tar: 1, tar: 1 }, outputs: { asphalt: 2 } },
wood_tar: { massPerUnitKg: 1, terminal: 'consumed' },
// + remove 'wood_tar' from NON_STORED_OUTPUTS (economy.ts:618)
// + keep producer forceRun; update §2.6 throttle guard in economy.test.ts

// (c) non-recipe sink — placement-cost add + retag
platform_constructor.placementCost += { asphalt: 100 };
asphalt: { /* mass */ terminal: 'gameplay-sink' };
Gate commands (per task / per phase)
# ledger
grep -cE "terminal: '?\`?expansion-hook" src/recipes.ts

# gate tests
npx vitest run src/recipes.test.ts src/mass-balance.test.ts src/economy.test.ts

# deadlock — bootstrap-reachable chains (byproduct loops, placement order)
timeout 590 python3 -u scripts/bootstrap_test.py | grep -E "BLOCKED|Full target build placed"

# deadlock — non-bootstrap chains (ocean, mid-tier)
npx tsx scripts/dump_recipe_graph.ts > /tmp/graph.json
python3 scripts/recipe_closure_check.py | grep -cE "CYCLE"   # expect 0
05

Risks & mitigations

Risk
Sev
Mitigation
Byproduct re-stall: removing a resource from NON_STORED_OUTPUTS gives it a capped bin; a full bin re-gates its producer to net 0 (the original §2.6 throttle bug).
HIGH
Keep every byproduct producer forceRun so it ignores the output cap and the surplus clamps (vents); the consumer draws from the stored bin. Re-point the §2.6 guard test at the still-vented set.
Bootstrap deadlock from gating a producer behind a component upstream of its own output (the P3 lesson).
HIGH
Every chain-introducing task runs the faithful planner (bootstrap-reachable) or the closure screen (non-bootstrap) before commit; planner must print Full target build placed, screen must report 0 cycles.
Mass-balance fails on non-unit-mass resources (mercury, helium_3, heavy fuels) when authoring refine recipes.
MED
Balance against actual massPerUnitKg; fall back to RECIPE_SPECULATIVE only for genuine fantasy chemistry (e.g. fusion-seed transmutation), never to dodge arithmetic.
Placement-cost add on an early (T1) building delays it until a late product exists, perturbing bootstrap order.
LOW
Add ocean/coastal/paving costs only to T2+ buildings off the producer's critical path; re-run the planner (costs affect affordability/order).
Test-file type errors hide from tsc -b (it excludes *.test.ts), e.g. a deleted/renamed ResourceId in a literal-pinning test.
LOW
Run the full vitest --project client suite, not just tsc; grep tests for literal toEqual input/output assertions after editing a recipe.
06

Decisions locked

Minimize new buildings ✓
Prefer recipe-input adds + resolveRecipe variants; add a building only where no existing host fits (~4 new: tar_refinery, vapor_condenser, cryo_reliquefier, mill_scale_sinter, plus the amalgamator).
XP weight inherits producer tier ✓
New consumer recipes' produced resource gets XP_WEIGHT matching the producing tier (T0=1, T1=3, T2=10, …); existing products keep their weight.
Byproduct loops only if no deadlock ✓
All six prototyped in the planner and proven deadlock-free (Full target build placed in 2449.58 days, 258 buildings, no BLOCKED).
Endgame fuels = gameplay-sink ✓
antimatter_propellant and plasma_charge are already consumed as placement costs — retag only.
