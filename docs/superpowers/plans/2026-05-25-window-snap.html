Window-snap — reject degenerate panel layouts (1-task plan)

For agentic workers REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development. Fresh kimi subagent + two-stage review (spec → code-quality) + nit-fix pass. Trailer literal on commits: Co-Authored-By: Kimi K2.6 <noreply@kimi.com>.

Goal: stop panels from being saved or loaded at degenerate (e.g. 1×1) dimensions. Floor = 80×40.
Architecture: two-layer guard in src/window-manager.ts. Load-time validation in parseLayoutBlob drops below-floor entries silently (cleansing existing degenerate saves on next page load). Capture-time guard in promoteToFree clamps the persisted rect against the resize-floor defaults (preventing future degenerate saves).
Tech stack: Vite 5 + TypeScript strict + vitest. Strict + noUncheckedIndexedAccess + noUnusedLocals.
Source of truth: design spec at docs/superpowers/specs/2026-05-25-window-snap-design.html (v1, user-approved 2026-05-25).
Locked design choices (do NOT deviate):
  • Floor constants: MIN_PANEL_W = 80, MIN_PANEL_H = 40.
  • Below-floor entries in parseLayoutBlob are dropped SILENTLY (no warn, no throw) — matches existing pattern for malformed entries.
  • NO schema bump. NO new fields. NO behaviour change for above-floor saves.
  • Promote-time clamp uses the resolved minWidth/minHeight from MakeDraggableOptions (defaults 200/120), NOT the new constants — those defaults are the user-resize minimum, more generous than the rejection floor.

Step 1. In src/window-manager.ts, add the floor constants right after the existing LAYOUT_STORAGE_KEY export (around line 18). The exports must be reachable from the test file:
/** Minimum width / height (px) for a persisted panel layout entry. Entries
 *  below either floor are dropped on load and refused at capture time —
 *  prevents the degenerate "1×1 invisible panel" state observed in v1 saves
 *  (settlement-panel, routes-panel) that originated when promoteToFree
 *  captured a placeholder-CSS-sized panel before any user resize.
 *  Floor chosen well below the 200×120 MakeDraggableOptions resize minimum
 *  (so legitimate small saves still pass) and well above the 1×1 bug. */
export const MIN_PANEL_W = 80;
export const MIN_PANEL_H = 40;
Step 2. In parseLayoutBlob (around lines 71-77 currently), tighten the per-entry dimension check. Find the existing validation block (search for w <= 0) and replace the w/h floor conditions:
// BEFORE:
if (
  typeof x !== 'number' || !Number.isFinite(x) ||
  typeof y !== 'number' || !Number.isFinite(y) ||
  typeof w !== 'number' || !Number.isFinite(w) || w <= 0 ||
  typeof h !== 'number' || !Number.isFinite(h) || h <= 0 ||
  typeof zRank !== 'number' || !Number.isFinite(zRank)
) continue;

// AFTER:
if (
  typeof x !== 'number' || !Number.isFinite(x) ||
  typeof y !== 'number' || !Number.isFinite(y) ||
  typeof w !== 'number' || !Number.isFinite(w) || w < MIN_PANEL_W ||
  typeof h !== 'number' || !Number.isFinite(h) || h < MIN_PANEL_H ||
  typeof zRank !== 'number' || !Number.isFinite(zRank)
) continue;
Note: only the w and h floors change. The other type guards stay identical. The continue silently drops the malformed entry — consistent with the existing behaviour for entries failing any other guard.

function promoteToFree(
  panel: HTMLElement,
  id: string,
  rect: DOMRect,
  minWidth: number,
  minHeight: number,
): void {
  if (panel.classList.contains('ri-free')) return;
  const flooredW = Math.max(rect.width, minWidth);
  const flooredH = Math.max(rect.height, minHeight);
  panel.style.position = 'fixed';
  panel.style.left = rect.left + 'px';
  panel.style.top = rect.top + 'px';
  panel.style.right = '';
  panel.style.bottom = '';
  panel.style.width = flooredW + 'px';
  panel.style.height = flooredH + 'px';
  panel.style.transform = '';
  panel.classList.add('ri-free');
  setPanelFree(id, true);
  updatePanelLayout(id, {
    x: rect.left, y: rect.top, w: flooredW, h: flooredH,
  });
}

Step 4. In src/window-manager.test.ts, add a new describe block testing the load-time floor. Place it next to the existing parseLayoutBlob tests (search for that name in the file):
import {
  parseLayoutBlob,
  MIN_PANEL_W,
  MIN_PANEL_H,
} from './window-manager.js';
// (Add MIN_PANEL_W + MIN_PANEL_H to the existing import block; don't duplicate.)

describe('parseLayoutBlob — degenerate-dimension floor', () => {
  it('rejects an entry with w=1, h=1 (the observed bug state)', () => {
    const raw = {
      v: 1,
      panels: {
        bad: { x: 79, y: 0, w: 1, h: 1, zRank: 294 },
      },
      globalZCounter: 0,
    };
    const result = parseLayoutBlob(raw);
    expect(result).not.toBeNull();
    expect(result!.panels.bad).toBeUndefined();
  });

  it('accepts an entry exactly at the floor (w=MIN_PANEL_W, h=MIN_PANEL_H)', () => {
    const raw = {
      v: 1,
      panels: {
        edge: { x: 0, y: 0, w: MIN_PANEL_W, h: MIN_PANEL_H, zRank: 0 },
      },
      globalZCounter: 0,
    };
    const result = parseLayoutBlob(raw);
    expect(result).not.toBeNull();
    expect(result!.panels.edge).toBeDefined();
    expect(result!.panels.edge!.w).toBe(MIN_PANEL_W);
    expect(result!.panels.edge!.h).toBe(MIN_PANEL_H);
  });

  it('rejects an entry one below either floor', () => {
    const raw = {
      v: 1,
      panels: {
        underW: { x: 0, y: 0, w: MIN_PANEL_W - 1, h: MIN_PANEL_H, zRank: 0 },
        underH: { x: 0, y: 0, w: MIN_PANEL_W, h: MIN_PANEL_H - 1, zRank: 0 },
      },
      globalZCounter: 0,
    };
    const result = parseLayoutBlob(raw);
    expect(result).not.toBeNull();
    expect(result!.panels.underW).toBeUndefined();
    expect(result!.panels.underH).toBeUndefined();
  });

  it('keeps valid entries when invalid ones are present in the same blob', () => {
    const raw = {
      v: 1,
      panels: {
        good: { x: 12, y: 12, w: 300, h: 200, zRank: 5 },
        bad: { x: 79, y: 0, w: 1, h: 1, zRank: 99 },
      },
      globalZCounter: 5,
    };
    const result = parseLayoutBlob(raw);
    expect(result).not.toBeNull();
    expect(result!.panels.good).toBeDefined();
    expect(result!.panels.bad).toBeUndefined();
    expect(result!.globalZCounter).toBe(5);
  });
});
