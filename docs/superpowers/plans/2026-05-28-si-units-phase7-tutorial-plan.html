Helper bodies

export function currentStep(world: World): TutorialStep | null {
  const ts = world.tutorialState ?? { completed: [], current: null };
  const completed = new Set(ts.completed);
  for (const step of TUTORIAL_STEPS) {
    if (completed.has(step.id)) continue;
    if (step.triggerCondition(world)) return step;
  }
  return null;
}

export function checkDismissals(world: World): ObjectiveId[] {
  const ts = world.tutorialState ?? { completed: [], current: null };
  const completed = new Set(ts.completed);
  const dismissed: ObjectiveId[] = [];
  for (const step of TUTORIAL_STEPS) {
    if (completed.has(step.id)) continue;
    if (step.dismissalCondition(world)) dismissed.push(step.id);
  }
  return dismissed;
}

export function markCompleted(world: World, id: ObjectiveId): void {
  world.tutorialState = world.tutorialState ?? { completed: [], current: null };
  if (!world.tutorialState.completed.includes(id)) {
    world.tutorialState.completed.push(id);
  }
}

export function skipAll(world: World): void {
  world.tutorialState = {
    completed: TUTORIAL_STEPS.map(s => s.id),
    current: null,
  };
}

export function restart(world: World): void {
  world.tutorialState = { completed: [], current: null };
}

Predicate-helper module

Steps 1–32 reference small predicate helpers (hasBuilding, invSeen, attemptedBuild, …). Implement them at the top of tutorial.ts rather than scattering inline expressions into each step. Each helper takes (world, …args) and returns boolean. Implementation notes:

  • hasBuilding(w, kinds) walks w.islands and returns true if any island has a placed building whose def.id matches any kind in the array. Constructed (not under-construction) only.
  • invSeen(w, r) — any island has ever had > 0 of r. Implement against a per-resource cumulative-seen flag if it exists on Phase 6's world model; otherwise approximate with w.islands.some(i => (i.state.inventory[r] ?? 0) > 0) and accept that an item the player burns through before the next poll won't trigger. Per spec §05 polling frequency is implementer's choice — once-per-game-second is sufficient resolution for any reasonable burn rate.
  • attemptedBuild(w, defId) — set by the placement-UI rejection path. Phase 5 wiring records this on a world.recentBuildAttempts: Set<BuildingDefId> with a 5-second TTL. If the field doesn't exist on Phase 6's world, this is a Phase 5 add (see §05 below).
  • hasAdjacentHeat(w, defId) — for a given building kind, check at least one placement has an adjacent tile occupied by a heat-emitting building. Reuse Phase 4's heat-graph helper if it exists; otherwise inline a simple 4-neighbour scan.
  • hasAdjacentSameType(w) — any two same-def.id buildings share an edge. Walk every island once.
  • stepCompleted(w, id, ttlMs) — true if id entered completed at least ttlMs ago. Used as a soft-dismiss for "hint stays visible 30 s then disappears" steps.

Module shape

// src/tutorial-ui.ts
import { currentStep, markCompleted } from './tutorial';
import type { World } from './world';

let hintEl: HTMLElement | null = null;
let renderedStepId: string | null = null;

export function refreshTutorialHint(world: World): void {
  const step = currentStep(world);
  if (!step) {
    if (hintEl) {
      hintEl.remove();
      hintEl = null;
      renderedStepId = null;
    }
    return;
  }
  if (step.id === renderedStepId) return;  // avoid re-render thrash
  if (!hintEl) {
    hintEl = document.createElement('div');
    hintEl.className = 'tutorial-hint';
    document.body.appendChild(hintEl);
    hintEl.addEventListener('click', () => {
      const cur = currentStep(world);
      if (cur) {
        markCompleted(world, cur.id);
        refreshTutorialHint(world);
      }
    });
  }
  hintEl.innerHTML = `
    <div class="tut-num">${step.id}</div>
    <div class="tut-mech">${step.mechanic}</div>
    <div class="tut-text">${step.hint}</div>
    ${step.expectedAction ? `<div class="tut-action">${step.expectedAction}</div>` : ''}
    <div class="tut-hint-dismiss">click to dismiss</div>
  `;
  renderedStepId = step.id;
}

Implementation notes

Useful context for the implementer that didn't rise to the level of a spec requirement.

  • Order matters. Phases 1 and 2 are independent of Phase 3+ and can land before any tutorial code is written. Doing them first means a working game with rev-9 starter exists before the tutorial wires in — the reachability test in Phase 7 needs both to evaluate.
  • Phase 3 is the largest single commit. ~350 LOC of mostly literal-table data (the 32 steps). Resist the urge to split per step — they're more readable as a single array.
  • Hot-path graph identity for polling. refreshTutorialHint(world) walks TUTORIAL_STEPS linearly; the guard at the top of each step's predicate (if (completed.has(step.id)) continue;) means a fully-completed tutorial costs 32 Set lookups + 0 predicate evaluations, basically free.
  • Don't export helpers from tutorial.ts beyond the spec's surface. Predicate helpers (hasBuilding, invSeen, …) stay module-private. Phase 6 tests use the public surface (currentStep, checkDismissals, etc.); never reach into private helpers from a test.
  • Step 8 (adjacency buff) and 11 (construction time) use stepCompleted(w, id, ttlMs) as their dismissal. This is a soft-dismiss for "read-and-move-on" hints with no enforcement criterion. The TTL is the hint visibility duration; the player can dismiss manually by clicking before it expires.
  • Step 17 (drones) and 18 (lighthouse) similarly use soft dismiss. The strict dismissal (droneRoutes.size > 0 / discoveredIslandCount(w) > startingDiscovered) is the satisfying-condition; the soft TTL is the fallback so the hint doesn't sit forever if the player ignores it.
  • Persistence schema does NOT bump. Per spec §05.1, tutorialState is already declared at persistence.ts:145 by an earlier phase. Phase 7 only populates it; no migration is required. Pre-Phase-7 saves load with tutorialState: undefined; currentStep treats this as empty completed.
  • Phase 9 fix-commits are NOT optional. Even if every checklist passes, log a "Phase 9 playtest verification" empty-ish commit (or amend the Phase 8 confirmation message) so the git history shows the playtest gate completed. This is the closure marker for the entire 7-phase SI-units rework.
  • Dev tab discipline. Per AGENTS.md, the systemd service serves dist/, not source. Every iteration: npm run build → reload tab. Do NOT restart robot-islands-dev.service for code changes.
