Saltwater-cell battery bootstrap — Implementation Plan

Plan summary
Goal: Make battery_bank (T2, 5 kWh) actually buildable from a fresh home-Plains start by introducing a new T2 component saltwater_cell (crafted at a new T1 building cell_press from home-buildable inputs only) and rewriting battery_bank.placementCost from { battery: 4, wire: 3, steel_beam: 1 } → { saltwater_cell: 4, wire: 3, steel_beam: 1 }. The T3 battery token stays as-is for capacitor_bank / flywheel_array / singularity_battery_factory.
Architecture: Pure additive resource-catalog + building-catalog work plus a one-line placementCost reroute. New surface: 1 ResourceId · 1 BuildingDefId · 1 RecipeId. Two atomic commits: Commit A ships the supply side (resource + def + recipe), Commit B reroutes battery_bank's cost. Each commit compiles and passes the suite independently.
Tech Stack: TypeScript (strict, noUncheckedIndexedAccess, noUnusedLocals), Vite 5, PixiJS 8, vitest. Pure layer only — no PixiJS, no rendering touches.
Spec: docs/superpowers/specs/2026-05-26-saltwater-cell-battery-bootstrap-design.html (v1) · Audit: docs/superpowers/audits/2026-05-26-battery-bank-chain.html · Reference plans: 2026-05-23-coal-rebalance.html (closest-precedent single-feature shape).
For agentic workers: REQUIRED SUB-SKILL: use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (- [ ]) syntax for tracking. Mechanical additive-catalog work — well-suited to a Kimi K2.6 subagent brief; the implementing agent's own co-author trailer goes on each commit.
