For agentic workers REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development. Fresh kimi subagent per task + two-stage review (spec compliance reviewer → code quality reviewer) per task. Trailer literal on commits: Co-Authored-By: Kimi K2.6 <noreply@kimi.com>.

Goal: fix three drone-scan bugs preceding the skill-tree rebalance — tier-keyed scan radius (T1=2, T2=4, T3=8); unified buffer-and-flush so out-of-antenna-range corridor cells survive to flush on antenna entry or return; no-antenna case handled by the same buffer.
Architecture: add per-tier radius table and scanBuffer: Set<string> on the Drone interface; rewrite the per-tick reveal block in advanceDrones so every corridor cell joins the buffer unconditionally and a separate flush step drains it when the drone is in any antenna's range OR has just reached 'returned' status. The T5 island-discovery dark-mode buffer (darkModeDiscoveries) stays; its flush logic moves into the same place so cells + islands flush together.
Tech stack: Vite 5 + TypeScript strict + PixiJS 8 + vitest. Strict + noUncheckedIndexedAccess + noUnusedLocals.
Source of truth: design spec at docs/superpowers/specs/2026-05-25-drone-scan-rework-design.html (v1, user-approved 2026-05-25). T6 scan radius locked at 16 per the design's open question (proposed value confirmed by approval).
Locked design choices (do NOT deviate):
  • Per-tier scan radius: T1 = 2, T2 = 4, T3 = 8, T4 = 0 (pulse-only, unused), T5 = 12 (unchanged), T6 = 16.
  • Single unified buffer-and-flush across T1–T6. The existing T5 dark-mode island branch consolidates into the same flush.
  • Flush triggers: (a) drone position in any antenna range at tick end; (b) status transition to 'returned'. Status 'lost' drops the buffer with the drone.
  • Schema bump v12 → v13. scanBuffer serialised as sorted array (mirrors revealedCells); migration defaults [] on every drone.
  • NO unrelated refactors. NO changes to T4 pulse, weather rolls, fuel mechanics, route logic, or skill-effect catalog. Pure drone-scan repair.

/** Drain a drone's scanBuffer + darkModeDiscoveries into world state.
 *  Idempotent — clears both buffers after draining. Called when the drone
 *  enters antenna range OR reaches 'returned' status. */
function flushDroneBuffers(
  d: Drone,
  world: WorldState,
  newlyDiscoveredIslandIds: string[],
): number {
  let cellsAdded = 0;
  for (const k of d.scanBuffer) {
    if (world.revealedCells.has(k)) continue;
    world.revealedCells.add(k);
    cellsAdded++;
  }
  d.scanBuffer.clear();
  for (const disc of d.darkModeDiscoveries) {
    const isl = world.islands.find((i) => i.id === disc.islandId);
    if (isl && !isl.discovered && !isl.populated) {
      isl.discovered = true;
      newlyDiscoveredIslandIds.push(isl.id);
    }
  }
  d.darkModeDiscoveries = [];
  return cellsAdded;
}

  it('reveals corridor cells when origin has no antennas, drone returns safely', () => {
    // Construct a minimal world: one populated origin island, NO antenna
