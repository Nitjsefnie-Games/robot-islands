Solve pipeline

Where the fixpoint sits in computeRates. The loop wraps pass 2.5 (solve gates) ⇄ pass 3 (aggregate watts → pf); pass 4 then realizes base × g × pf at the converged pair. A unified cable component (or pre-pass override) fixes pf and skips the loop.

Dashed clay = the fixpoint loop (Pass 2.5 ⇄ Pass 3 driven by solveBrownoutFactor). Off-brownout it runs once. The cable pre-pass runs the same loop over a component's member islands, sharing one pf.

03

Key code — the leaf

The whole coupling is one scalar: gates are deterministic in pf, so the leaf iterates pf alone. Pure, callback-driven, unit-testable in isolation.

src/flow-power-fixpoint.ts
export function solveBrownoutFactor(
  evalAtPf: (pf: number) => PowerSample,
  opts?: { maxIters?: number; epsilon?: number },
): BrownoutFixpointResult {
  const maxIters = opts?.maxIters ?? BROWNOUT_FIXPOINT_MAX_ITERS;
  const eps = opts?.epsilon ?? BROWNOUT_FIXPOINT_EPSILON;

  // No-brownout fast path: surplus at full power ⇒ pf=1 is the fixed
  // point — ONE eval, byte-identical to today for non-brownout pools.
  const pfFull = pfOf(evalAtPf(1));
  if (pfFull >= 1 - eps) return { powerFactor: 1, converged: true, iterations: 1 };

  let pf = pfFull;
  for (let i = 1; i <= maxIters; i++) {
    const target = pfOf(evalAtPf(pf));
    const next = (pf + target) / 2;        // damp the decreasing map
    if (Math.abs(next - pf) < eps)
      return { powerFactor: next, converged: true, iterations: i + 1 };
    pf = next;
  }
  return { powerFactor: pf, converged: false, iterations: maxIters + 1 };
}
