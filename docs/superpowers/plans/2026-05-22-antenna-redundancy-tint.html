Antenna redundancy tint — Implementation Plan

Plan summary
Goal: Tint an antenna's overlay rust-leaning when its full coverage disc is covered by the union of all OTHER antennas' discs — a visual hint that the antenna is safe to demolish.
Architecture: A pure helper isAntennaRedundant(test, others, samples=24) in antenna.ts samples the test antenna's perimeter (24 points + centre) and reuses the existing pointInSignalRange union test. antenna-overlay.ts calls it in its existing per-antenna render loop, swapping the colour to REDUNDANT_COLOR = 0xd6928a when the predicate is true. No new files; pure/render separation preserved.
Tech Stack: TypeScript (strict, noUncheckedIndexedAccess, noUnusedLocals), Vite 5, PixiJS 8, vitest.
Spec: docs/superpowers/specs/2026-05-22-antenna-redundancy-tint-design.html
For agentic workers: REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (- [ ]) syntax for tracking.

// extend the existing import from './antenna.js'
// (the file already pulls SignalRange / computeSignalRanges / pointInSignalRange)

import { isAntennaRedundant, REDUNDANT_SAMPLES } from './antenna.js';

describe('isAntennaRedundant', () => {
  it('a lone antenna is never redundant', () => {
    const test: SignalRange = { cx: 0, cy: 0, radius: 80 };
    expect(isAntennaRedundant(test, [])).toBe(false);
  });

  it('an antenna fully inside one bigger antenna is redundant', () => {
    // test radius 50 centred at origin; other radius 200 also at origin → fully covers.
    const test: SignalRange = { cx: 0, cy: 0, radius: 50 };
    const others: SignalRange[] = [{ cx: 0, cy: 0, radius: 200 }];
    expect(isAntennaRedundant(test, others)).toBe(true);
  });

  it('two co-located identical antennas — each redundant relative to the other', () => {
    // closed-disc test: perimeter samples lie ON the other's perimeter → inside.
    const a: SignalRange = { cx: 0, cy: 0, radius: 80 };
    const b: SignalRange = { cx: 0, cy: 0, radius: 80 };
    expect(isAntennaRedundant(a, [b])).toBe(true);
    expect(isAntennaRedundant(b, [a])).toBe(true);
  });

  it('an antenna with a clearly-uncovered perimeter is not redundant', () => {
    // test centred at (0, 0) radius 100; the only "other" sits far to the right.
    // The test's left perimeter (~ x = -100) is nowhere near the other's disc.
    const test: SignalRange = { cx: 0, cy: 0, radius: 100 };
    const others: SignalRange[] = [{ cx: 500, cy: 0, radius: 50 }];
    expect(isAntennaRedundant(test, others)).toBe(false);
  });

  it('union of two flanking antennas covers a test neither one alone covers', () => {
    // test radius 80 at origin. Each "other" (r=100, offset ±40 on x) covers
    // only its half — the (40,0) disc misses the test's left perimeter
    // (point (-80,0) is dist 120 > 100), the (-40,0) disc misses the right.
    // Their UNION covers every perimeter sample. The test must return true
    // only if isAntennaRedundant uses the union (pointInSignalRange) and not
    // any single-other check.
    const test: SignalRange = { cx: 0, cy: 0, radius: 80 };
    const others: SignalRange[] = [
      { cx: 40, cy: 0, radius: 100 },
      { cx: -40, cy: 0, radius: 100 },
    ];
    expect(isAntennaRedundant(test, others)).toBe(true);
  });

  it('exports REDUNDANT_SAMPLES = 24', () => {
    expect(REDUNDANT_SAMPLES).toBe(24);
  });
});

Step 3: Implement isAntennaRedundant in src/antenna.ts

Append after the existing pointInSignalRange (~line 77). Re-uses pointInSignalRange for the union test; no new geometric primitive.

/** Default perimeter sample count for the redundancy check. 24 evenly
 *  spaced perimeter points + the centre = 25 union tests per call.
 *  Chord length 2·r·sin(π/24) ≈ 0.26·r — tighter than 16 at modest cost. */
export const REDUNDANT_SAMPLES = 24;

/** True iff the test antenna's coverage disc is fully covered by the
 *  union of `others`' discs. Approximated by sampling N perimeter points
 *  + the centre and checking each against the existing union helper.
 *  Pure — no PixiJS, no world mutation. A visual hint, not a removal-
 *  safety guarantee (see spec §02 for honest error accounting). */
export function isAntennaRedundant(
  test: SignalRange,
  others: ReadonlyArray<SignalRange>,
  samples: number = REDUNDANT_SAMPLES,
): boolean {
  if (others.length === 0) return false;
  if (!pointInSignalRange(others, test.cx, test.cy)) return false;
  for (let k = 0; k < samples; k++) {
    const theta = (2 * Math.PI * k) / samples;
    const x = test.cx + test.radius * Math.cos(theta);
    const y = test.cy + test.radius * Math.sin(theta);
    if (!pointInSignalRange(others, x, y)) return false;
  }
  return true;
}

Step 2: Branch the per-antenna render

In the existing per-antenna draw inside rebuild() (the loop that draws SIGNAL_COLOR at antenna-overlay.ts:47-50), wrap the colour pick in the redundancy test. The loop currently calls computeSignalRanges(populated) once and iterates. Change to:

// inside rebuild(), replacing the existing forEach/for loop body:
const ranges: SignalRange[] = computeSignalRanges(populated);
for (let i = 0; i < ranges.length; i++) {
  const r = ranges[i]!;
  // `others` = ranges without index i. New array per antenna; N is small.
  const others = ranges.slice(0, i).concat(ranges.slice(i + 1));
  const colour = isAntennaRedundant(r, others) ? REDUNDANT_COLOR : SIGNAL_COLOR;
  // keep the rest of the per-antenna draw exactly as it was — only `colour`
  // replaces the previous SIGNAL_COLOR reference in the .fill / .stroke chain.
  gfx
    .circle(r.cx * TILE_PX, r.cy * TILE_PX, r.radius * TILE_PX)
    .fill({ color: colour, alpha: RING_FILL_ALPHA })
    .stroke({ color: colour, width: 1, alpha: RING_STROKE_ALPHA });
}

The constants RING_FILL_ALPHA / RING_STROKE_ALPHA / TILE_PX are whatever the existing code already uses — keep their existing names. If the existing loop uses a .forEach or different variable names, preserve those patterns and only swap the colour pick.

Notes

Pure / render separation preserved. Task 1 lives entirely in antenna.ts (pure layer, no PixiJS), tested via antenna.test.ts without a renderer. Task 2 confines render-layer changes to antenna-overlay.ts. drones.ts and orbital.ts (the other readers of antenna ranges) are untouched.
Performance is fine. The naïve nested loop is O(N · (N · 25)) per refresh() with N = antenna count. For typical N ≤ 50 that's ≤ 62 500 cheap union tests per rebuild; refresh() fires only on world-layer rebuild, not every frame. No caching needed.
T6 / spaceport interaction is out of scope. A T6 antenna may be geometrically redundant for drone-telemetry but still required for groundStationCommRange (orbital.ts:367-368). The spec defers this to a later refinement — the visual hint here only reflects drone-telemetry redundancy.
Allocation in the per-antenna loop. ranges.slice().concat() allocates O(N) per antenna. Acceptable on a rebuild path; if profiling later flags it, swap to an index-skipping wrapper around pointInSignalRange rather than re-shaping the array.
