Mass Building Actions

Goal Select many buildings on one island (shift-click + shift-drag box) and act on them at once — destroy, upgrade, group-move, enable/disable, ignore-cap — by batching the existing per-building gateway operations. No new gateway method, no schema bump, no migration. Executed via subagent-driven development on branch feat/mass-building-actions.

02

Load-bearing pure logic

The two helpers most likely to be done wrong: the upgrade planner (slot + running-inventory math) and the group-relocate validator (sibling-position interaction).

src/mass-actions.ts — planMassUpgrade
export function planMassUpgrade(state, selectedIds) {
  const free = (parallelBuildSlots(state) - inProgressBuildCount(state))
    + (queuedBuildSlots(state) - queuedBuildCount(state));
  if (free <= 0) return [];
  const ids = new Set(selectedIds);
  const candidates = state.buildings
    .filter((b) => ids.has(b.id) && (b.constructionRemainingMs ?? 0) <= 0 && b.queued !== true)
    .sort((a,b) => (a.floorLevel ?? 0) - (b.floorLevel ?? 0) || (a.id < b.id ? -1 : 1));
  const running = { ...state.inventory };
  const plan = [];
  for (const b of candidates) {
    if (plan.length >= free) break;
    const cost = upgradeCost(BUILDING_DEFS[b.defId], topUpgradeLevel(state, b) + 2);
    if (Object.keys(affordabilityShortfall(running, cost)).length > 0) continue; // skip
    for (const [r,n] of Object.entries(cost)) running[r] = (running[r] ?? 0) - n;
    plan.push(b.id);
  }
  return plan;
}

Validate against a spec with all moving members pre-applied/removed; do member-overlap separately on post-move footprints. Flagged in Task 3; implementer documents the chosen approach.
Plan vs execution drift: planMassUpgrade preview count diverges from what applyUpgrade actually enqueues (it self-checks slots/affordability).
MED
Execution loops applyUpgrade in plan order and trusts ITS gates — break on queue-full, skip other failures. Planner drives only order + the "n fit" label.
Strict-mode breakage from staged callback stubs in Task 5 (noUnusedLocals/Parameters).
LOW
Implement each callback body in its own task; Task 5 ships only the Destroy path wired. Typecheck gate every task.
